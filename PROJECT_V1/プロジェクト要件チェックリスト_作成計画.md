**作成日**: 2025-12-15
**バージョン**: 1.1
**目的**: 開発工程.mdの各フェーズに対して、用途・目的ベースの質問形式チェックリストを作成する
**更新履歴**:
- v1.0: 初版作成
- v1.1: 技術スタック選択質問を追加 (クラウドプラットフォーム、開発言語、フレームワーク、データベース等)

---

## 1. 計画概要

### 1.1. 目的
- 具体的なツール名ではなく、**必要な機能・用途**を問う形式のチェックリストを作成
- プロジェクトの特性に応じて必要な作業項目を選択できるようにする
- 各開発工程(docs/開発工程.mdの## セクション)ごとに質問を構造化

### 1.2. 対象ファイル
- **入力**: [docs/開発工程.md](../docs/開発工程.md)
- **出力**: `PROJECT_V1/開発工程別ツール選択システム_計画書.md` (書き直し版)

### 1.3. 設計方針
- ツール名は記載せず、**用途・目的・機能**で質問
- 技術スタック選択質問を追加 (AWS/Azure/GCP、言語選択等)
- チェックボックス形式 (`- [ ]`) で選択可能
- 各質問には簡潔な説明を付与
- 技術選択肢は具体的なサービス/ツール名を列挙 (選択の参考として)

## 2. チェックリスト構成設計

### 2.1. 階層構造

```markdown
# プロジェクト要件チェックリスト

## 1. 企画プロセス
### 1.1. [サブセクション名]
- [ ] 質問1 (用途の説明)
- [ ] 質問2 (用途の説明)

## 2. 要件定義
### 2.1. 業務分析
- [ ] ビジネスプロセスの可視化は必要ですか? (業務フロー図作成)
- [ ] 業務機能の構成整理は必要ですか? (機能分解)
...
```

### 2.2. 質問設計パターン

#### パターン1: 機能必要性
```markdown
- [ ] [機能名]は必要ですか? (具体的な用途)
```

例:
```markdown
- [ ] 負荷テストは必要ですか? (同時アクセス時の動作確認)
```

#### パターン2: 成果物必要性
```markdown
- [ ] [成果物名]の作成は必要ですか? (目的)
```

例:
```markdown
- [ ] ユースケース図の作成は必要ですか? (機能要件の可視化)
```

#### パターン3: スコープ確認
```markdown
- [ ] [対象範囲]は含まれますか? (説明)
```

例:
```markdown
- [ ] 外部システム連携は含まれますか? (API/ファイル連携)
```

#### パターン4: 技術選択
```markdown
- [ ] [技術カテゴリ]の選択: 〇〇 / △△ / ××
```

例:
```markdown
- [ ] クラウドプラットフォームの選択: AWS / Azure / GCP / オンプレミス
- [ ] 開発言語の選択: C# / TypeScript / Java / Python / Go / その他
```
