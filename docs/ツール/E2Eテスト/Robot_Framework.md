# Robot Framework

## 概要

Robot Framework はキーワード駆動のテスト自動化フレームワークである。読みやすいテキスト形式でシナリオを記述でき、Python ベースで拡張可能なため、E2E テストから受け入れテストまで幅広く利用される。

## 主な特徴

| 項目 | 内容 |
|------|------|
| キーワード駆動 | 非エンジニアにも読みやすい記述 |
| 高い拡張性 | Python ライブラリで機能追加が容易 |
| 豊富な連携 | SeleniumLibrary など主要ライブラリを利用可能 |
| レポート出力 | 実行ログと HTML レポートを自動生成 |
| 再利用性 | 共通キーワードで保守性を向上 |

## インストールとセットアップ

公式URL:
- [Robot Framework 公式](https://robotframework.org/)
- [User Guide](https://robotframework.org/robotframework/latest/RobotFrameworkUserGuide.html)
- [Robot Framework GitHub](https://github.com/robotframework/robotframework)

セットアップの要点:
1. Python 環境を準備して `robotframework` を導入する。
2. 必要に応じて SeleniumLibrary など対象技術のライブラリを追加する。
3. `tests`、`resources`、`keywords` の分割方針を先に決める。

## 基本的な使い方

1. まず業務シナリオをキーワードベースで短く記述する。
2. 共通処理は Resource ファイルに切り出して再利用する。
3. タグで smoke/regression を分け、実行対象を制御する。
4. 失敗時ログとレポートを確認してシナリオを改善する。

最小コマンド:
- テスト実行: `robot tests/`

## メリット

- テスト手順を読みやすく共有しやすい
- 非エンジニアもレビューに参加しやすい
- ライブラリ連携で適用範囲を広げられる
- レポートが標準で充実している

## デメリット

- キーワード設計が曖昧だと保守性が下がる
- 大規模化時に構成ルールが必須
- 複雑ロジックは Python 実装が必要になる

## 他ツールとの比較

| ツール | 主な対象 | 特徴 |
|------|------|------|
| Robot Framework | キーワード駆動 E2E | 可読性と拡張性のバランスが良い |
| Cucumber | BDD | 業務仕様の振る舞い記述に強い |
| Playwright | Web E2E | 実装者向けに高速・高機能 |
| Selenium | Web 自動化 | 言語資産と実績が豊富 |

## ベストプラクティス

### 1. キーワード命名を統一

- 業務用語ベースで命名し、役割を明確化する
- 実装詳細を隠蔽して読みやすさを維持する

### 2. レイヤー分離を徹底

- テストケース、キーワード、ライブラリを分離する
- UI 変更影響をキーワード層で吸収する

### 3. タグ運用を標準化

- 実行時間と目的でタグを分ける
- リリース前に必要タグだけを確実に実行する

## まとめ

1. ** 可読管理 ** : キーワード駆動により、テスト手順を読みやすく共有しやすい形で管理できる。
2. ** 段階拡張 ** : ライブラリ拡張を使って、Web/API など複数領域の自動化を段階的に広げられる。
3. ** 保守向上 ** : レイヤー分離とタグ運用を徹底すると、E2E スイートの保守性を高めやすい。
