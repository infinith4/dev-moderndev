# IntelliJ IDEA

## 概要

IntelliJ IDEA は JetBrains が提供する統合開発環境である。Java/Kotlin を中心に高度なコード解析、リファクタリング、デバッグ、テスト支援を提供し、JVM 系開発の生産性を高めやすい。

## 料金

| プラン | 内容 |
|------|------|
| Community | 無料（Apache 2.0、機能限定） |
| Ultimate | 有料（Web/DB/企業向け機能を含む） |

## 主な特徴

| 項目 | 内容 |
|------|------|
| 強力な補完 | 文脈理解に基づく高精度補完 |
| リファクタ支援 | 安全な大規模変更を実施しやすい |
| JVM 親和性 | Java/Kotlin/Gradle/Maven 連携が強い |
| プラグイン拡張 | 言語・フレームワーク対応を追加可能 |
| デバッグ統合 | 実行・テスト・診断を IDE で一元化 |

## 主な機能

### コーディング機能

| 機能 | 説明 |
|------|------|
| コード補完 | 型情報と文脈に応じた提案 |
| クイックフィックス | 問題箇所の即時修正 |
| リファクタリング | Rename、Extract、Move など |
| 検索/ナビゲーション | 定義参照と構造把握を高速化 |

### 品質・デバッグ機能

| 機能 | 説明 |
|------|------|
| 静的解析 | 潜在バグや規約違反を検知 |
| デバッガ | ブレークポイントと変数監視 |
| テスト実行 | JUnit 等を IDE から実行 |
| カバレッジ | テスト網羅率を確認 |

### 運用機能

| 機能 | 説明 |
|------|------|
| Git 統合 | コミット、差分確認、マージ支援 |
| Database ツール | DB 接続とクエリ検証（Ultimate） |
| Framework 支援 | Spring などの開発支援（Ultimate） |
| 設定共有 | プロジェクト設定を標準化しやすい |

## インストールとセットアップ

公式URL:
- [IntelliJ IDEA 公式](https://www.jetbrains.com/idea/)
- [IntelliJ IDEA Docs](https://www.jetbrains.com/help/idea/)
- [JetBrains Toolbox](https://www.jetbrains.com/toolbox-app/)

セットアップの要点:
1. Community または Ultimate を選んで導入する。
2. JDK とビルドツール（Gradle/Maven）を設定する。
3. チーム標準のコードスタイルと inspections を適用する。

## 基本的な使い方

1. プロジェクトを開き、依存解決が完了することを確認する。
2. 実行構成（Run/Debug Configuration）を定義する。
3. 日常開発では補完、クイックフィックス、リファクタを活用する。
4. 変更後は IDE からテスト実行して差分を確認する。

## メリット

- JVM 系開発で高い生産性を得やすい
- リファクタと静的解析が強力
- 大規模コードベースでも追跡しやすい
- JetBrains 製品群との連携がしやすい

## デメリット

- 高機能ゆえに学習コストがある
- 環境によってはメモリ使用量が大きい
- Ultimate の利用には費用が発生する

## 他ツールとの比較

| ツール | 主な対象 | 特徴 |
|------|------|------|
| IntelliJ IDEA | Java/Kotlin 開発 | 静的解析とリファクタが強い |
| VS Code | 汎用エディタ | 軽量・拡張中心 |
| Visual Studio | .NET/C++ | Microsoft スタックに強い |
| Eclipse | Java 開発 | OSS ベースで実績がある |

## ベストプラクティス

### 1. inspections を標準化

- チーム共通ルールをプロファイル化する
- 警告無視を最小化する

### 2. 実行構成を共有

- 開発・テスト構成をテンプレート化する
- 新規参加者の立ち上がりを短縮する

### 3. リファクタを積極活用

- 手動置換より IDE の安全機能を優先する
- 変更後はテスト実行で即確認する
