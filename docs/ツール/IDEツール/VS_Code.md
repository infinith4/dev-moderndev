# VS Code

## 概要

VS Code は Microsoft が提供する軽量で拡張性の高いコードエディタである。多言語開発、デバッグ、Git 連携、拡張機能を一つの環境で扱えるため、個人開発からチーム開発まで幅広く利用される。

## 料金

| プラン | 内容 |
|------|------|
| 本体 | 無料 |
| 拡張機能 | 多くは無料（有料拡張あり） |

## 主な特徴

| 項目 | 内容 |
|------|------|
| 軽量性 | 起動が速く環境構築しやすい |
| 拡張性 | Marketplace で機能追加が容易 |
| 多言語対応 | Web、Python、Go、Java などに対応 |
| Git 統合 | 変更確認、コミット、差分確認を IDE 内で実行 |
| Dev Container | 開発環境をコンテナ化して再現可能 |

## 主な機能

### コーディング機能

| 機能 | 説明 |
|------|------|
| IntelliSense | 補完と型情報の提示 |
| Lint/Format | 保存時整形や静的解析連携 |
| スニペット | 定型コードを再利用 |
| マルチカーソル | 複数箇所を同時編集 |

### デバッグ機能

| 機能 | 説明 |
|------|------|
| ブレークポイント | ステップ実行で原因調査 |
| 変数監視 | 実行時状態を可視化 |
| launch 設定 | 実行構成をプロジェクト管理 |
| ターミナル統合 | 実行・テストを IDE 内で完結 |

### 運用機能

| 機能 | 説明 |
|------|------|
| 拡張管理 | チーム標準拡張を共有 |
| 設定同期 | 複数端末で環境を統一 |
| Remote 開発 | SSH/WSL/Container へ接続 |
| タスク自動化 | build/test/lint をタスク化 |

## インストールとセットアップ

公式URL:
- [VS Code 公式](https://code.visualstudio.com/)
- [VS Code Docs](https://code.visualstudio.com/docs)
- [Marketplace](https://marketplace.visualstudio.com/vscode)

セットアップの要点:
1. VS Code をインストールする。
2. 使用言語の拡張機能を追加する。
3. ワークスペース設定でフォーマット・lint ルールを統一する。

## 基本的な使い方

1. プロジェクトをフォルダ単位で開く。
2. 推奨拡張機能を導入し、補完・lint・デバッグを有効化する。
3. `tasks.json` と `launch.json` を作成し、実行手順を標準化する。
4. Git 変更を IDE 内で確認し、差分レビューを行う。

## メリット

- 導入しやすく拡張で柔軟に育てられる
- 多言語開発に対応しやすい
- 開発環境の標準化を進めやすい
- Dev Container で再現性を高めやすい

## デメリット

- 拡張を入れすぎると動作が重くなりやすい
- 設定が分散するとチーム標準が崩れやすい
- 大規模プロジェクトではフル IDE より機能不足を感じる場合がある

## 他ツールとの比較

| ツール | 主な対象 | 特徴 |
|------|------|------|
| VS Code | 汎用開発エディタ | 軽量で拡張性が高い |
| IntelliJ IDEA | JVM 中心の統合開発 | 静的解析とリファクタ支援が強い |
| Visual Studio | .NET / Windows 開発 | 高機能 IDE とデバッグに強い |
| Neovim | キーボード中心開発 | 高速でカスタマイズ性が高い |

## ベストプラクティス

### 1. 拡張機能を標準化

- 必須拡張と任意拡張を分ける
- 推奨拡張をプロジェクトに明示する

### 2. 設定をリポジトリ管理

- `.vscode/settings.json` に共通設定を置く
- フォーマッタと lint ルールを統一する

### 3. 実行タスクを定義

- build/test/lint をタスク化する
- ローカル実行手順を固定して属人化を防ぐ
