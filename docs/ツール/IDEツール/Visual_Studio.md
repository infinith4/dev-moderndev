# Visual Studio

## 概要

Visual Studio は Microsoft が提供するフル機能 IDE である。.NET と C++ 開発を中心に、デバッグ、テスト、プロファイリング、設計支援を統合しており、大規模開発でも生産性を維持しやすい。

## 料金

| プラン | 内容 |
|------|------|
| Community | 個人・一部組織向け無料 |
| Professional / Enterprise | 商用向け有料（機能・サポート強化） |

## 主な特徴

| 項目 | 内容 |
|------|------|
| フル IDE | 開発からテストまで統合環境で実行可能 |
| .NET 親和性 | ASP.NET、Blazor、MAUI などに強い |
| 高度デバッグ | ブレークポイント、診断ツール、プロファイラ |
| 設計支援 | リファクタ、コード分析、テスト生成支援 |
| 企業運用 | エンタープライズ向け機能が豊富 |

## インストールとセットアップ

公式URL:
- [Visual Studio 公式](https://visualstudio.microsoft.com/)
- [Visual Studio Docs](https://learn.microsoft.com/visualstudio/)
- [Visual Studio Downloads](https://visualstudio.microsoft.com/downloads/)

セットアップの要点:
1. 必要ワークロード（.NET、Web、C++ など）を選んでインストールする。
2. チーム標準の analyzers と formatter を導入する。
3. テスト実行とデバッグ設定を共通化する。

## 基本的な使い方

1. ソリューションを開き、依存関係を復元する。
2. 主要起動プロジェクトを設定してローカル実行する。
3. ブレークポイントを使って挙動を確認する。
4. テストエクスプローラーで回帰テストを実行する。

## メリット

- 高度なデバッグと診断機能が使える
- .NET 開発で高い生産性を得やすい
- 大規模ソリューションを管理しやすい
- 企業向け機能が整っている

## デメリット

- 比較的重く高スペック環境が必要
- 初期インストールサイズが大きい
- 料金プランによって利用機能が分かれる

## 他ツールとの比較

| ツール | 主な対象 | 特徴 |
|------|------|------|
| Visual Studio | .NET / C++ 開発 | フル IDE と診断機能が強い |
| VS Code | 汎用エディタ | 軽量で拡張中心 |
| IntelliJ IDEA | JVM 開発 | Java/Kotlin 支援が強い |
| Rider | .NET 開発 | クロスプラットフォーム IDE |

## ベストプラクティス

### 1. ワークロードを最小化

- 必要な開発対象だけインストールする
- 不要コンポーネントを避けて更新負荷を減らす

### 2. 規約を IDE に組み込む

- analyzer と formatter を共通化する
- 警告レベルをチームで統一する

### 3. テスト実行を日常化

- 変更前後で対象テストを実行する
- 失敗時の再現手順を記録する
