# Slack

## 概要

Slack は、チャンネル型のビジネスコミュニケーションツールである。メッセージ、スレッド、ファイル共有、通知連携を一元化し、開発・運用・業務連絡を同じワークスペースで管理できる。

## 料金

| プラン | 内容 |
|------|------|
| Free | 小規模利用向けの基本機能 |
| Pro / Business+ | 履歴、管理、セキュリティ機能を拡張 |
| Enterprise Grid | 大規模組織向けの統制・運用機能 |

## 主な特徴

| 項目 | 内容 |
|------|------|
| チャンネル中心運用 | テーマ・案件単位で会話を整理しやすい |
| 非同期コミュニケーション | スレッドで議論を分離しやすい |
| 外部連携 | GitHub、Jira、Google Drive などと連携可能 |
| 通知制御 | メンション、DND、リマインダーで通知を調整 |
| 検索性 | 履歴・添付ファイルを横断検索可能 |
| 自動化 | Workflow や Webhook で定型作業を省力化 |

## 主な機能

### コミュニケーション機能

| 機能 | 説明 |
|------|------|
| チャンネル | プロジェクト単位で会話を整理 |
| スレッド | 本文を汚さず議論を継続 |
| DM / グループDM | 少人数連絡に利用 |
| メンション | 対象メンバーへ明示的に通知 |

### 連携・自動化機能

| 機能 | 説明 |
|------|------|
| App 連携 | GitHub/Jira などのイベント通知を集約 |
| Incoming Webhook | 外部システムからメッセージ投稿 |
| Workflow Builder | 申請・定期通知などを自動化 |
| Slack API / Bot | 独自コマンドや業務連携を実装 |

### 管理・運用機能

| 機能 | 説明 |
|------|------|
| 権限管理 | ワークスペースやチャンネル権限を制御 |
| 監査・ログ | プランに応じて運用ログを管理 |
| 通知ポリシー | 重要通知と雑多通知を分離 |
| ステータス管理 | 勤務状況や応答可否を共有 |

## インストールとセットアップ

公式URL:
- [Slack 公式サイト](https://slack.com/)
- [Slack Help Center](https://slack.com/help)
- [Slack API](https://api.slack.com/)
- [Slack App Directory](https://slack.com/apps)

セットアップの要点:
1. ワークスペースを作成し、チャンネル命名規則を決める。
2. メンバー招待と権限ロールを設定する。
3. GitHub/Jira など必要な連携アプリを追加する。
4. 通知ルール（メンション、DND、運用時間）を定義する。

## 基本的な使い方

1. テーマ別にチャンネルを作成し、会話を分離する。
2. 課題議論はスレッドで進め、要点だけ本線に残す。
3. 重要イベントは連携アプリで通知を自動投稿する。
4. `/remind` や Workflow で定期タスクを自動化する。
5. 決定事項はピン留め・ブックマークで参照しやすくする。

最小運用例:
- 開発通知: `#dev-notify` に CI 結果を投稿
- 問い合わせ: `#support` にテンプレート付き投稿を運用

## メリット

- 情報共有をチャンネル単位で整理しやすい。
- 開発ツール連携により通知を集約しやすい。
- 非同期運用を前提に、リモート環境でも回しやすい。

## デメリット

- チャンネル乱立で情報発見性が落ちやすい。
- 通知設計が不十分だとノイズ過多になりやすい。
- 高度な統制機能は上位プランが前提になる。

## 他ツールとの比較

| ツール | 主な対象 | 特徴 |
|------|------|------|
| Slack | 開発・業務コミュニケーション | 連携アプリと自動化が豊富 |
| Microsoft Teams | Microsoft 365 統合 | Office/Entra 環境と親和性が高い |
| Discord | コミュニティ運用 | 音声運用やカジュアルな会話に強い |
| Google Chat | Google Workspace 連携 | Gmail/Meet と統合しやすい |

## ベストプラクティス

### 1. チャンネル設計を標準化

- 目的別に `proj-`、`ops-`、`team-` など接頭辞を統一する。
- 決定事項は固定チャンネルへ集約する。

### 2. 通知ルールを明確化

- `@channel` の利用条件を決める。
- 監視通知は専用チャンネルに隔離する。

### 3. 自動化を段階導入

- まずは CI/CD と課題管理通知から連携する。
- 効果が高い定型作業のみ Workflow 化する。

## まとめ

1. **情報整理** : チャンネル中心の運用で、チーム内コミュニケーションを整理しやすい。
2. **通知集約** : 外部ツール連携により、開発・運用イベントの通知を集約しやすい。
3. **運用設計** : 命名規則と通知設計を先に決めると、長期運用での混乱を抑えやすい。
