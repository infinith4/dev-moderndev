# Zephyr for Jira

## 概要

Zephyr for Jira は、Jira 上でテストケース管理と実行管理を行うテスト管理ツールである。Jira の課題・スプリント運用と連動し、要件とテスト進捗を同一画面で追跡しやすい。

## 料金

| プラン | 内容 |
|------|------|
| Zephyr for Jira | 有料（プラン別） |
| Trial | 無料トライアルあり |

## 主な特徴

| 項目 | 内容 |
|------|------|
| Jira統合 | Jira課題とテスト資産を連動管理 |
| テストサイクル | スプリント・リリース単位で実行管理 |
| 不具合連携 | 失敗結果からBug課題へ接続可能 |
| 可視化レポート | 実行率、合格率、カバレッジを表示 |
| 自動化連携 | JUnit/Cucumber等の結果取込に対応 |
| アジャイル適合 | スプリント運用と合わせやすい |

## 主な機能

### テスト管理機能

| 機能 | 説明 |
|------|------|
| テストケース作成 | 手順、期待結果、前提条件を管理 |
| テストサイクル | 実行単位をリリース/スプリントで管理 |
| 実行結果登録 | Pass/Fail/Blocked などを記録 |
| エビデンス添付 | 失敗時の証跡を添付 |

### トレーサビリティ機能

| 機能 | 説明 |
|------|------|
| 要件リンク | Story と Test を紐付け |
| 不具合リンク | テスト失敗を Bug に連携 |
| カバレッジ表示 | 要件単位の網羅率を可視化 |
| 進捗ダッシュボード | テスト状況を定点監視 |

### 連携・自動化機能

| 機能 | 説明 |
|------|------|
| API連携 | 実行結果を外部から登録可能 |
| CI連携 | Jenkins/GitHub Actions 等と連携 |
| BDD連携 | Cucumber シナリオとの連携が可能 |
| レポート出力 | 監査/報告用の出力が可能 |

## インストールとセットアップ

公式URL:
- [Zephyr 公式サイト](https://www.getzephyr.com/)
- [Zephyr Scale](https://www.getzephyr.com/products/zephyr-scale)
- [Zephyr Squad](https://www.getzephyr.com/products/zephyr-squad)
- [Documentation](https://support.smartbear.com/zephyr-scale-cloud/docs/)

セットアップの要点:
1. Jira Marketplace から Zephyr を導入する。
2. プロジェクトでテスト課題タイプとサイクルを有効化する。
3. 要件リンクと不具合連携の運用ルールを定める。
4. CI から結果取込を設定し、手動結果と統合する。

## 基本的な使い方

1. テストケースを作成して手順と期待結果を登録する。
2. テストサイクルを作成し、対象ケースを割り当てる。
3. 実行結果を記録し、失敗時に不具合課題を紐付ける。
4. ダッシュボードで進捗と品質指標を確認する。
5. 自動テスト結果をAPI/連携プラグインで反映する。

最小実行例:
- 手動実行: Test Cycle 上で結果更新
- 自動連携: JUnit 結果を API 経由で取り込み

## メリット

- Jira運用に統合し、導入ハードルを下げやすい。
- テスト進捗と要件カバレッジを可視化しやすい。
- 手動と自動の結果を一体運用しやすい。

## デメリット

- Jira依存のため、単体導入はできない。
- 大規模運用では権限・構成設計が必要。
- ライセンス費用とプラン差分の管理が必要。

## CI/CD での使用

CI ではテスト結果を Zephyr に取り込み、手動実行結果と統合する。スプリント終了時にサイクル単位で結果を締める運用にすると、品質判定を標準化しやすい。

## 他ツールとの比較

| ツール | 主な対象 | 特徴 |

| Zephyr for Jira | Jira統合テスト管理 | スプリント連動の運用に強い |

| TestRail | 独立テスト管理 | Jira非依存で柔軟運用可能 |

## ベストプラクティス

### 1. サイクル運用を固定

- スプリント/リリース単位でサイクル命名を統一する。
- 締め処理のルールを決める。

### 2. リンク運用を標準化

- Story↔Test↔Bug のリンク規則を明文化する。
- 欠落リンクを定期的に監査する。

### 3. 自動取込を継続

- CI から結果連携を自動化する。
- 手動結果との差分をレビューで確認する。
