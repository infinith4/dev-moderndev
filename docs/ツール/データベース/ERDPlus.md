# ERDPlus

## 概要

ERDPlus は、ブラウザ上で ER 図や関係スキーマを作成できる無料ツールである。シンプルな操作で概念設計を可視化し、SQL のひな形生成まで行えるため、学習用途や小規模プロジェクトで使いやすい。

## 料金

| プラン | 内容 |
|------|------|
| ERDPlus | 無料 |
| 商用利用 | 利用規約の範囲で可能（組織ポリシー確認は必要） |

## 主な特徴

| 項目 | 内容 |
|------|------|
| ブラウザ完結 | インストール不要で利用可能 |
| ER モデリング | エンティティ、属性、関係を図式化 |
| 関係スキーマ化 | ER 図から関係モデルへ変換 |
| SQL 生成 | テーブル定義のひな形を出力可能 |
| 学習向けUI | 操作がシンプルで初学者にも扱いやすい |

## 主な機能

### モデリング機能

| 機能 | 説明 |
|------|------|
| ER 図作成 | エンティティとリレーションを配置 |
| 属性設定 | 主キー、多値属性、導出属性などを設定 |
| カーディナリティ | 1:1、1:N、N:M を表現 |
| 関係スキーマ変換 | 概念モデルを論理モデルに変換 |

### 出力機能

| 機能 | 説明 |
|------|------|
| SQL 出力 | CREATE TABLE 文のひな形を生成 |
| 画像出力 | 図を PNG で保存 |
| モデル共有 | 作成モデルをレビュー用に共有しやすい |
| 学習資料化 | 授業・レビュー資料として利用可能 |

### 設計支援機能

| 機能 | 説明 |
|------|------|
| 正規化検討 | 中間エンティティで多対多を解消しやすい |
| 命名統一 | テーブル・カラム命名の確認に使える |
| 設計レビュー | 業務担当者と設計認識を合わせやすい |
| 初期DDL準備 | 実装前のたたき台を準備しやすい |

## インストールとセットアップ

公式URL:
- [ERDPlus 公式サイト](https://erdplus.com/)

セットアップの要点:
1. ブラウザで ERDPlus にアクセスする。
2. 図のタイプ（ER Diagram / Relational Schema）を選ぶ。
3. プロジェクト命名と設計ルール（命名規則）を先に決める。
4. 必要に応じて SQL 出力形式を選択する。

## 基本的な使い方

1. エンティティを配置し、主要属性と主キーを定義する。
2. リレーションを引き、カーディナリティを設定する。
3. 多対多は中間エンティティで解消する。
4. 関係スキーマへ変換し、外部キーを確認する。
5. SQL を出力して実装前レビューに回す。

最小実行例:
- 入力: 顧客、注文、商品の ER 図
- 出力: 関係スキーマ + SQL ひな形

## メリット

- 無料で導入でき、教育・検討用途に使いやすい。
- 視覚的に設計を整理し、レビュー効率を上げやすい。
- ER 図から SQL ひな形を得られ、初期実装を進めやすい。

## デメリット

- エンタープライズ向けの高度機能は限定的。
- チーム共同編集や厳密な変更管理には別ツールが必要。
- 大規模スキーマでは管理性が下がりやすい。

## 他ツールとの比較

| ツール | 主な対象 | 特徴 |
|------|------|------|
| ERDPlus | 学習/小規模ER設計 | 無料でシンプル、SQL ひな形生成が可能 |
| MySQL Workbench | MySQL 設計/運用 | 設計から管理まで統合 |
| DBeaver | マルチDB運用 | 接続・クエリ中心で運用しやすい |
| draw.io | 汎用図作成 | 自由度は高いがDB特化機能は限定的 |

## ベストプラクティス

### 1. 命名規則を先に決める

- テーブル名、主キー名、外部キー名を統一する。
- レビュー時の命名差分を減らす。

### 2. 段階的にモデル化する

- まず主要エンティティだけで概念図を作る。
- 詳細属性はレビュー後に追加する。

### 3. SQL 出力を過信しない

- 生成SQLはたたき台として扱う。
- インデックス、制約、型は実装環境で再調整する。
