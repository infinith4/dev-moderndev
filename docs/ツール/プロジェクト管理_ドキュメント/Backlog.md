# Backlog

## 概要

Backlog は、ヌーラボが提供するプロジェクト管理・課題管理ツールである。課題管理、Wiki、Git/SVN ホスティング、ガントチャートを統合し、開発チームと非エンジニアが共通基盤で協業しやすい。

## 料金

| プラン | 内容 |
|------|------|
| Free | 個人・小規模向け基本機能 |
| Starter | 小規模チーム向けの管理機能 |
| Standard | 中規模チーム向けの拡張機能 |
| Premium | 大規模チーム向けの高度機能 |
| Enterprise | 組織向け統制・セキュリティ機能 |

## 主な特徴

| 項目 | 内容 |
|------|------|
| 課題管理 | タスク、バグ、要望を体系的に管理 |
| Git/SVN 統合 | ソースコード管理を同一ツールで運用可能 |
| ガントチャート | スケジュールと依存関係を可視化 |
| Wiki 機能 | プロジェクト知識をチーム内で共有 |
| 日本語対応 | 日本語 UI と国内サポートを提供 |
| シンプル設計 | 非エンジニアでも操作しやすい |

## 主な機能

### 課題管理機能

| 機能 | 説明 |
|------|------|
| 課題登録/管理 | 種別、優先度、担当者、期限を設定 |
| カスタム属性 | プロジェクトに合わせて項目を追加 |
| 親子課題 | 課題を階層化して作業を分割 |
| 検索/フィルタ | 条件指定で課題を抽出 |

### 開発支援機能

| 機能 | 説明 |
|------|------|
| Git ホスティング | リポジトリ管理とプルリクエスト |
| SVN ホスティング | Subversion リポジトリを提供 |
| Wiki | Markdown 対応のドキュメント管理 |
| ファイル共有 | プロジェクト単位でファイルを管理 |

### 可視化・連携機能

| 機能 | 説明 |
|------|------|
| ガントチャート | 計画と進捗を視覚的に管理 |
| バーンダウンチャート | スプリント進捗を可視化 |
| Webhook/API | 外部ツールとの連携に対応 |
| 通知機能 | メール・チャット連携で更新を共有 |

セットアップの要点:
1. スペースを作成し、メンバーを招待する。
2. プロジェクトを作成し、課題種別とカテゴリを定義する。
3. マイルストーンとバージョンで期間管理を整理する。
4. Git リポジトリや Wiki を必要に応じて有効化する。

## 基本的な使い方

1. プロジェクトを作成し、メンバーと権限を設定する。
2. 課題を登録して担当者・期限・優先度を割り当てる。
3. ガントチャートまたはボードで進捗を確認する。
4. 完了した課題をクローズし、マイルストーン達成を管理する。
5. 週次で未完了課題と遅延状況をレビューする。

最小運用例:
- 計画: マイルストーン設定 → 課題登録
- 実行: 課題更新 → 日次確認

## メリット

- 日本語 UI と国内サポートで導入障壁が低い。
- 課題管理と Git ホスティングを一元運用しやすい。
- ガントチャートで計画と進捗を可視化しやすい。
- 非エンジニアを含むチーム全体で利用しやすい。

## デメリット

- 大規模アジャイル運用には機能が不足する場合がある。
- カスタマイズ性は Jira 等と比較すると限定的。
- 上位プランでないと利用できない機能がある。

## チーム運用での使用

課題管理とガントチャートを軸に計画と実行を管理し、Wiki でプロジェクト知識を蓄積する運用が有効である。Git 連携で課題とコミットを紐付けると、変更追跡がしやすくなる。

## 他ツールとの比較

| ツール | 主な用途 | 特徴 |
|------|------|------|
| Backlog | 課題管理/Git 統合 | 日本語対応とシンプル UI が強み |
| Jira | 課題管理/アジャイル | ワークフローと拡張エコシステムが豊富 |
| Redmine | OSS 課題管理 | 軽量で導入しやすい |
| OpenProject | OSS プロジェクト管理 | セルフホストとガント管理に強い |

## ベストプラクティス

### 1. 課題管理ルールを標準化

- 課題種別、優先度、カテゴリの運用ルールを統一する。
- 完了条件を明確にしてクローズ基準を固定する。

### 2. マイルストーンで期間管理

- リリース単位でマイルストーンを設定する。
- 定期的に進捗を確認し、遅延を早期検知する。

### 3. Git 連携を活用

- コミットメッセージに課題キーを含めて紐付ける。
- プルリクエストでコードレビューを運用する。
