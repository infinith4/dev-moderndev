# Docusaurus

## 概要

Docusaurus は、技術ドキュメントサイトを構築するための静的サイトジェネレーターである。Markdown ベースでドキュメントを管理し、バージョン管理や検索、国際化に対応したドキュメントポータルを運用しやすい。

## 主な特徴

| 項目 | 内容 |
|------|------|
| Markdown 中心 | ドキュメントをテキストで管理可能 |
| バージョン管理 | バージョン別ドキュメントを公開可能 |
| 国際化対応 | 多言語サイトを構築しやすい |
| React ベース | テーマやページ拡張がしやすい |
| 検索/ナビ対応 | サイドバー・検索導線を整備しやすい |
| 静的配信 | CDN 配信しやすく運用コストを抑えやすい |

## 主な機能

### ドキュメント構築機能

| 機能 | 説明 |
|------|------|
| Docs ページ | Markdown から文書ページを生成 |
| Blog ページ | リリースノートや更新情報を管理 |
| Sidebar 管理 | 階層ナビゲーションを定義 |
| Front Matter | メタ情報や表示制御を設定 |

### サイト運用機能

| 機能 | 説明 |
|------|------|
| Versioning | 過去バージョン文書を保持 |
| i18n | 多言語コンテンツを分離管理 |
| Theme/Plugin | 検索・分析・表示拡張に対応 |
| Static Build | 配信用ファイルを出力 |

### 開発連携機能

| 機能 | 説明 |
|------|------|
| ローカルプレビュー | 開発中に即時確認可能 |
| Git 連携 | 文書差分を PR でレビュー可能 |
| CI/CD 統合 | ビルドと公開を自動化しやすい |
| リンク検証 | ビルド時にリンク不整合を検知可能 |

## インストールとセットアップ

公式URL:
- [Docusaurus 公式サイト](https://docusaurus.io/)
- [Getting Started](https://docusaurus.io/docs)
- [Deployment](https://docusaurus.io/docs/deployment)
- [GitHub](https://github.com/facebook/docusaurus)

セットアップの要点:
1. Node.js 環境を準備する。
2. `create-docusaurus` で初期プロジェクトを作成する。
3. `docs/` と `sidebars.js` を整備する。
4. CI で `build` 成功を公開条件にする。

## 基本的な使い方

1. Markdown でドキュメントを追加する。
2. サイドバーへページ構成を反映する。
3. `start` でローカルプレビューを確認する。
4. `build` で静的ファイルを生成する。
5. 生成物をホスティングへデプロイする。

最小運用例:
- 開発: `npm run start`
- 公開ビルド: `npm run build`

## メリット

- 文書をコード管理でき、変更追跡がしやすい。
- バージョン別ドキュメント運用に強い。
- 拡張性が高く、企業ドキュメント基盤に合わせやすい。
- 静的配信で公開運用を軽量化しやすい。

## デメリット

- React/Node.js 前提の初期学習が必要。
- サイト設計が未整理だとナビ構造が複雑化しやすい。
- 高度なカスタマイズは保守コストが増えやすい。

## CI/CD での使用

PR で文書変更をレビューし、CI で Docusaurus のビルドとリンク検証を実施する運用が有効である。main マージ後に自動デプロイすることで、公開反映までの手順を標準化しやすい。

## ベストプラクティス

### 1. 情報設計を先に固定

- カテゴリとサイドバー構造を先に定義する。
- 文書テンプレートで記述粒度を統一する。

### 2. バージョン運用を明確化

- 対象バージョンと公開期間を決める。
- 古い版の更新ルールを文書化する。

### 3. 公開フローを自動化

- CI でビルド失敗時はマージ不可にする。
- デプロイとリンクチェックを定期実行する。
