# Jira

## 概要

Jira は Atlassian が提供する課題管理・プロジェクト管理ツールである。アジャイルボード、カスタムワークフロー、自動化機能を備え、ソフトウェア開発の計画・実行・追跡を一元管理しやすい。

## 料金

| プラン | 内容 |
|------|------|
| Free | 小規模チーム向け基本機能 |
| Standard | チーム運用向けの管理機能を拡張 |
| Premium | 高度な計画・自動化・管理機能を強化 |
| Enterprise | 大規模組織向け統制・セキュリティ機能 |

## 主な特徴

| 項目 | 内容 |
|------|------|
| 課題管理 | タスク、バグ、ストーリーを体系的に管理 |
| アジャイル対応 | スクラムボード・カンバンボードを標準提供 |
| カスタムワークフロー | チームの業務フローに合わせて状態遷移を定義可能 |
| Atlassian 連携 | Confluence、Bitbucket 等と統合しやすい |
| 自動化機能 | ルールベースで課題操作を自動化 |
| レポート機能 | バーンダウン、ベロシティ等で進捗を可視化 |

## 主な機能

### 課題管理機能

| 機能 | 説明 |
|------|------|
| Issue 管理 | タスク、バグ、ストーリー、エピックを管理 |
| カスタムフィールド | 組織要件に合わせて項目を拡張 |
| 優先度/ラベル | 分類と優先順位付けを標準化 |
| 検索/フィルタ（JQL） | 条件指定で課題を柔軟に抽出 |

### アジャイル運用機能

| 機能 | 説明 |
|------|------|
| スクラムボード | スプリント計画と実行を管理 |
| カンバンボード | WIP 制限付きのフロー管理 |
| バックログ管理 | 優先順位付けとスプリント割当 |
| レポート | バーンダウン、ベロシティ、累積フロー図 |

### 管理・連携機能

| 機能 | 説明 |
|------|------|
| 自動化ルール | 状態変更・通知・割当を自動化 |
| 権限スキーム | プロジェクト別にアクセス制御 |
| REST API | 外部システムとデータ連携可能 |
| Marketplace | アドオンで機能拡張 |

セットアップの要点:
1. Cloud 利用か Data Center（セルフホスト）運用かを決める。
2. プロジェクト種別（スクラム/カンバン/チーム管理）を選択する。
3. ワークフローと課題タイプを業務に合わせて定義する。
4. 権限スキームと通知設定を整理する。

## 基本的な使い方

1. プロジェクトを作成し、メンバーを招待する。
2. バックログに課題（ストーリー/タスク/バグ）を登録する。
3. スプリント計画で課題を割り当て、ボードで進捗を管理する。
4. 完了条件を確認してスプリントをクローズする。
5. レポートでベロシティと残課題を振り返る。

最小運用例:
- 計画: バックログ整理 → スプリント計画
- 実行: ボード更新 → 日次スタンドアップ確認

## メリット

- アジャイル開発の計画・実行・振返りを一元管理しやすい。
- ワークフローのカスタマイズでチーム固有の業務フローに適合しやすい。
- Atlassian 製品群との連携で開発基盤を統合しやすい。
- JQL による柔軟な検索で課題の抽出・分析がしやすい。

## デメリット

- 設定項目が多く、初期設計に時間がかかりやすい。
- ワークフローやスキームが複雑化すると保守負荷が増える。
- 高度な機能は上位プランや有料アドオン前提になりやすい。

## チーム運用での使用

スクラム運用ではスプリント計画とバーンダウンチャートで進捗を可視化し、カンバン運用では WIP 制限とフロー効率を重視する。日次スタンドアップでボードを確認し、スプリントレビューでベロシティを振り返る構成が有効である。

## 他ツールとの比較

| ツール | 主な用途 | 特徴 |
|------|------|------|
| Jira | 課題管理/アジャイル | ワークフローと拡張エコシステムが豊富 |
| OpenProject | OSS プロジェクト管理 | セルフホストとガント管理に強い |
| Redmine | OSS 課題管理 | 軽量で導入しやすい |
| Azure DevOps | 組織統制 | Microsoft 連携とエンタープライズ管理に強い |

## ベストプラクティス

### 1. ワークフローを簡潔に設計

- 状態数を最小限にして運用負荷を下げる。
- 遷移条件と完了定義を明確にする。

### 2. バックログ管理を継続

- 定期的にバックログを整理して優先順位を更新する。
- エピックで大きな作業単位を可視化する。

### 3. 自動化で定型作業を削減

- ステータス変更時の通知・担当割当を自動化する。
- 古い課題の自動クローズルールを設定する。

2. **柔軟性** : ワークフローと自動化を活用すると、チーム固有の業務フローに適合しやすい。
3. **拡張性** : Atlassian 連携と Marketplace を組み合わせると、開発基盤全体を統合運用しやすい。
