# Miro

## 概要

Miro は、オンラインホワイトボード型のコラボレーションツールである。企画、要件整理、設計レビュー、振り返りなどをリアルタイムで実施でき、分散チームでも可視化中心の合意形成を進めやすい。

## 料金

| プラン | 内容 |
|------|------|
| Free | 基本機能を利用可能 |
| 有料プラン | ボード数拡張、管理機能、セキュリティ機能を提供 |
| Enterprise | 大規模組織向け統制機能を提供 |

## 主な特徴

| 項目 | 内容 |
|------|------|
| 無限キャンバス | 大規模な検討を1ボードで管理可能 |
| リアルタイム編集 | 複数人で同時に作業可能 |
| 豊富なテンプレート | ワークショップや計画作業を開始しやすい |
| コメント/投票 | 合意形成プロセスを支援 |
| 外部連携 | Jira、Confluence、Slack などと連携可能 |
| ファシリテーション支援 | タイマーやプレゼンモードを利用可能 |

## インストールとセットアップ

公式URL:
- [Miro 公式サイト](https://miro.com/)
- [Miro Help Center](https://help.miro.com/)
- [Miro Templates](https://miro.com/templates/)
- [Miro Apps](https://miro.com/apps/)

セットアップの要点:
1. ワークスペースを作成し、チームメンバーを招待する。
2. 権限ポリシー（編集/閲覧）を先に定義する。
3. 代表テンプレートを標準化してボード乱立を防ぐ。
4. Jira/Confluence 連携を必要な範囲で有効化する。

## 基本的な使い方

1. 会議目的に合うテンプレートを選択する。
2. 付箋や図形で情報を可視化する。
3. コメントと投票で論点を整理する。
4. 決定事項をフレーム単位でまとめる。
5. 結果を PDF/画像で共有し、他ツールへ反映する。

最小運用例:
- 会議前: テンプレート準備
- 会議中: 付箋整理 + 投票
- 会議後: 決定事項を出力・共有

## メリット

- リモート環境でも議論を可視化しやすい。
- テンプレート活用で会議準備を短縮しやすい。
- 非エンジニアを含む多職種連携に向いている。
- 成果物を他ツールへ連携しやすい。

## デメリット

- 運用ルールなしではボードが散在しやすい。
- 大規模ボードは閲覧性が低下しやすい。
- 高度な管理機能は有料プラン前提になりやすい。

## チーム運用での使用

要件定義やスプリント計画で Miro を使い、合意した内容を Jira やドキュメント基盤へ同期する運用が有効である。ワークショップ前後の手順をテンプレート化すると、会議品質を安定させやすい。

## ベストプラクティス

### 1. ボード設計を標準化

- 目的別テンプレートを定義する。
- 命名規則と保管場所を統一する。

### 2. 会議運営を定型化

- タイムボックスと投票手順を固定する。
- 決定事項と未決事項を明確に分離する。

### 3. 成果物連携を徹底

- 会議後に Jira/Docs へ反映する責任者を決める。
- ボードをアーカイブして履歴を残す。
