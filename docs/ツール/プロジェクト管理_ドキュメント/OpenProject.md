# OpenProject

## 概要

OpenProject は、オープンソースのプロジェクト管理ツールである。タスク管理、ガントチャート、アジャイルボード、工数管理、Wiki などを統合し、開発プロジェクトの計画と進捗を一元管理しやすい。

## 料金

| プラン | 内容 |
|------|------|
| Community Edition | 無料（OSS） |
| 有料プラン | 追加機能・サポートを提供 |
| 自社運用 | セルフホストで運用可能 |

## 主な特徴

| 項目 | 内容 |
|------|------|
| OSS ベース | セルフホスト運用に対応 |
| プロジェクト管理統合 | 課題、日程、工数を一元化 |
| ガントチャート | 計画と依存関係を可視化 |
| アジャイル対応 | スクラム/カンバン運用を支援 |
| 文書機能 | Wiki とドキュメント管理に対応 |
| 権限管理 | 役割別にアクセスを制御可能 |

## インストールとセットアップ

公式URL:
- [OpenProject 公式サイト](https://www.openproject.org/)
- [OpenProject Documentation](https://www.openproject.org/docs/)
- [Installation Guide](https://www.openproject.org/docs/installation-and-operations/installation/)
- [OpenProject GitHub](https://github.com/opf/openproject)

セットアップの要点:
1. SaaS 利用かセルフホスト運用かを決める。
2. プロジェクト種別ごとにワークフローを定義する。
3. ロールと権限を最初に整理する。
4. タスク種別・優先度・工数項目を標準化する。

## 基本的な使い方

1. プロジェクトを作成し、メンバーを割り当てる。
2. Work Package を登録して担当・期限を設定する。
3. ガントまたはボードで進捗を管理する。
4. 工数を入力し、負荷と実績を可視化する。
5. 週次で未完了タスクとリスクをレビューする。

最小運用例:
- 計画: マイルストーン + ガント設定
- 実行: Work Package 更新 + 工数入力

## メリット

- OSS で導入しやすく、セルフホスト要件に対応しやすい。
- タスク、スケジュール、工数を一元管理しやすい。
- 権限制御で組織運用へ適用しやすい。
- ドキュメント機能でプロジェクト知識を残しやすい。

## デメリット

- 初期設定（ワークフロー・権限設計）に時間がかかる。
- UI/運用が未整備だと入力負荷が高くなりやすい。
- 高度な統合要件では追加設計が必要。

## チーム運用での使用

開発計画ではガントで依存関係を明確化し、実行フェーズではボードと工数入力を日次運用する構成が有効である。定例レビューで KPI を確認すると、遅延検知を早めやすい。

## 他ツールとの比較

| ツール | 主な用途 | 特徴 |
|------|------|------|
| OpenProject | OSS プロジェクト管理 | セルフホストと統合管理に強い |
| Jira | 課題管理 | 拡張エコシステムが豊富 |
| Redmine | OSS 課題管理 | 軽量で導入しやすい |
| Taiga | アジャイル管理 | シンプルなスクラム運用向き |

## ベストプラクティス

### 1. 入力ルールを標準化

- Work Package の必須項目を固定する。
- 期限・担当・優先度の記入ルールを統一する。

### 2. レビューサイクルを固定

- 日次で進捗更新、週次で計画見直しを行う。
- 遅延タスクのエスカレーション基準を決める。

### 3. 権限とテンプレートを整備

- ロール別権限を最小権限で設計する。
- プロジェクトテンプレートで立上げを標準化する。
