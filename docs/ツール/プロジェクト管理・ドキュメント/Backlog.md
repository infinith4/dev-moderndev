# Backlog

## 概要

Backlog は、株式会社ヌーラボが開発・提供する日本発のプロジェクト管理・コラボレーションツールです。課題管理（Issue Tracking）、Git/Subversion によるバージョン管理、Wiki、ガントチャート、バーンダウンチャートを統合し、開発チームから非エンジニアまで直感的に使えるUI/UXを提供します。日本語に完全対応し、国内企業に最適化された機能により、170万人以上のユーザーに利用されています。

## 基本的な使い方

### 1. プロジェクトの作成

```
1. ホーム画面右上の「+」ボタンをクリック
2. 「プロジェクトを追加」を選択
3. プロジェクト情報を入力
   - プロジェクト名
   - プロジェクトキー（英数字）
   - 説明
4. 「追加」をクリック
```

### 2. 課題の登録

```
1. プロジェクトを選択
2. 「課題を追加」ボタンをクリック
3. 課題情報を入力
   - 種別: タスク/バグ/要望など
   - 件名: 課題のタイトル
   - 詳細: 課題の説明（Markdown可）
   - 担当者: 担当者を選択
   - 優先度: 高/中/低
   - マイルストーン: 対象バージョン
   - 期限日: 完了予定日
4. 「登録」をクリック
```

### 3. 課題の更新

```
1. 課題をクリックして詳細画面を開く
2. ステータスを変更
   - 未対応 → 処理中 → 処理済み → 完了
3. コメントを追加
   - @メンションで担当者に通知
   - ファイルを添付
4. 実績時間を記録（オプション）
5. 「更新」をクリック
```

### 4. Git リポジトリの利用

```bash
# リポジトリのクローン
git clone https://your-space.backlog.com/git/YOUR_PROJECT/repository.git

# 課題番号を含めたコミット（自動リンク）
git commit -m "PROJECT-123 ログイン機能の実装"

# プッシュ
git push origin main

# 課題 PROJECT-123 に自動的にコミットがリンクされる
```

### 5. プルリクエストの作成

```
1. Git → プルリクエスト → 「追加」
2. プルリクエスト情報を入力
   - タイトル
   - 説明
   - ベースブランチ
   - トピックブランチ
   - 担当者（レビュアー）
3. 課題を選択してリンク
4. 「登録」をクリック
```

### 6. Wiki の作成

```
1. Wiki タブを選択
2. 「ページを追加」をクリック
3. ページ情報を入力
   - 名前
   - 内容（Markdown）
   - 親ページ（階層化の場合）
4. ファイルを添付（必要に応じて）
5. 「追加」をクリック
```

## 課題管理の実践例

### スクラム開発

```
# スプリント管理
1. マイルストーンでスプリントを作成
   - Sprint 1 (2024-01-15 〜 2024-01-28)
   - Sprint 2 (2024-01-29 〜 2024-02-11)

2. ユーザーストーリーを課題として登録
   - 種別: タスク
   - 件名: ユーザーがログインできる
   - 詳細: As a user, I want to...
   - マイルストーン: Sprint 1

3. タスクを分解（子課題）
   - ログイン画面の作成
   - 認証APIの実装
   - セッション管理

4. バーンダウンチャートで進捗確認
   - 課題 → グラフ → バーンダウンチャート
```

### バグ管理

```
# バグトラッキング
1. バグ課題の登録
   - 種別: バグ
   - 件名: ログアウト後もセッションが残る
   - 優先度: 高
   - 再現手順を詳細に記載

2. カスタム属性で分類
   - 発見環境: 本番環境
   - ブラウザ: Chrome 120
   - 影響度: 中

3. Git ブランチで修正
   git checkout -b fix/PROJECT-456
   git commit -m "PROJECT-456 セッション削除処理の追加"

4. プルリクエストでレビュー
5. マージ後、課題を「完了」に変更
```

### リリース管理

```
# バージョン管理
1. バージョンの作成
   - 設定 → バージョン → 「追加」
   - v1.2.0 (リリース日: 2024-02-01)

2. リリース対象課題にバージョンを設定
   - 発見バージョン: 修正対象のバージョン
   - マイルストーン: リリース予定のバージョン

3. ガントチャートでスケジュール確認
4. リリースノートを Wiki で作成
5. リリース完了後、バージョンをリリース済みに変更
```

## API 連携

### REST API の使用

```bash
# APIキーの取得
個人設定 → API → 「登録」

# 課題一覧の取得
curl -X GET "https://your-space.backlog.com/api/v2/issues?apiKey=YOUR_API_KEY&projectId[]=PROJECT_ID"

# 課題の登録
curl -X POST "https://your-space.backlog.com/api/v2/issues?apiKey=YOUR_API_KEY" \
  -d "projectId=PROJECT_ID" \
  -d "summary=新しい課題" \
  -d "issueTypeId=1" \
  -d "priorityId=3"

# 課題の更新
curl -X PATCH "https://your-space.backlog.com/api/v2/issues/PROJECT-123?apiKey=YOUR_API_KEY" \
  -d "statusId=3" \
  -d "comment=処理完了しました"
```

### Python での使用

```python
import requests

API_KEY = "YOUR_API_KEY"
SPACE_URL = "https://your-space.backlog.com"

# 課題一覧の取得
def get_issues(project_id):
    url = f"{SPACE_URL}/api/v2/issues"
    params = {
        "apiKey": API_KEY,
        "projectId[]": project_id
    }
    response = requests.get(url, params=params)
    return response.json()

# 課題の作成
def create_issue(project_id, summary, issue_type_id, priority_id):
    url = f"{SPACE_URL}/api/v2/issues"
    params = {"apiKey": API_KEY}
    data = {
        "projectId": project_id,
        "summary": summary,
        "issueTypeId": issue_type_id,
        "priorityId": priority_id
    }
    response = requests.post(url, params=params, data=data)
    return response.json()

# 課題の更新
def update_issue(issue_key, status_id, comment):
    url = f"{SPACE_URL}/api/v2/issues/{issue_key}"
    params = {"apiKey": API_KEY}
    data = {
        "statusId": status_id,
        "comment": comment
    }
    response = requests.patch(url, params=params, data=data)
    return response.json()

# 使用例
issues = get_issues(12345)
for issue in issues:
    print(f"{issue['issueKey']}: {issue['summary']}")
```

## トラブルシューティング

### よくある問題と解決策

#### 1. Git プッシュエラー

```bash
# エラー: 認証失敗
# 解決策: APIキーまたはパスワードを確認

# HTTPSでのクローン
git clone https://your-space.backlog.com/git/PROJECT/repo.git

# 認証情報を保存
git config credential.helper store
```

#### 2. 課題が見つからない

```
原因: フィルターが適用されている
解決策:
1. 課題一覧画面で「フィルターをクリア」
2. または検索バーで課題キーを直接検索
```

#### 3. 通知が届かない

```
原因: 通知設定が無効
解決策:
1. 個人設定 → 通知 → 各項目を確認
2. スター（ウォッチ）を有効化
3. メールフィルターを確認
```

## まとめ

Backlog は、以下の場面で特に有用です:

1. **日本企業でのプロジェクト管理** - 日本語完全対応、直感的なUI
2. **開発チームとビジネスの連携** - 非エンジニアも使いやすい
3. **統合プラットフォーム** - 課題管理・Git・Wiki を一元管理
4. **小〜中規模チーム** - 柔軟な料金プラン、スムーズな導入

Jira や GitHub に比べてシンプルで、日本企業の文化や業務フローに最適化されたツールとして、多くの企業に採用されています。
