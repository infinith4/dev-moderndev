# Confluence

## 概要

Confluenceは、Atlassian社が開発したチーム向けドキュメント管理・ナレッジベースツールです。Wikiベースの協業プラットフォームとして、プロジェクトドキュメント、要件定義書、設計書、議事録、ナレッジ記事などを一元管理できます。JIRA、Trello、Slackなど他のツールとシームレスに統合し、チームの情報共有とコラボレーションを促進します。

## 主な機能

### 1. ページ・スペース管理
- **スペース**: プロジェクトやチームごとの独立した領域
- **ページ階層**: 親子関係でドキュメントを構造化
- **テンプレート**: 会議議事録、プロジェクト計画、要件定義書等

### 2. リアルタイム協業
- 同時編集機能
- インラインコメント・メンション
- ページウォッチ・通知

### 3. バージョン管理
- ページ履歴の自動保存
- 過去バージョンへの復元
- 差分表示

### 4. 強力な検索機能
- 全文検索
- フィルタ（スペース、ラベル、作成者）
- 関連コンテンツ提案

### 5. マクロ・拡張機能
- **Status Macro**: タスク状態表示
- **Jira Macro**: JIRA課題埋め込み
- **Code Block**: ソースコード表示
- **Table of Contents**: 目次自動生成
- **Gliffy Diagrams**: 図表作成

### 6. 権限管理
- スペースレベル・ページレベルでのアクセス制御
- 閲覧/編集/コメント権限の細かい設定
- ゲストアクセス対応

### 7. 外部ツール統合
- **Jira**: 課題との双方向リンク
- **Trello**: ボード埋め込み
- **Slack**: 通知・検索統合
- **Google Drive、OneDrive**: ファイル連携
- **GitHub**: コミット・PR情報表示

## 利用方法

### 基本的な使い方

1. **スペース作成**
   ```
   Spaces → Create Space
   → プロジェクト/チーム名を入力
   → テンプレート選択 or 空白スペース
   ```

2. **ページ作成**
   ```
   スペース内で Create → Blank Page
   → ページタイトル入力
   → 本文をエディタで作成
   → Publish
   ```

3. **テンプレート使用**
   ```
   Create → From Template
   → テンプレート選択（会議議事録/要件定義書等）
   → 内容をカスタマイズして保存
   ```

4. **テーブル追加**
   ```
   / → Table
   → 行・列数を指定
   → セル内容を入力
   ```

5. **Jira課題埋め込み**
   ```
   / → Jira Issue/Filter
   → Jira URLまたは課題キーを入力
   → 表示形式を選択
   ```

6. **コメント・メンション**
   ```
   ページ下部の Add Comment
   → @ユーザー名 でメンション
   → コメント投稿
   ```

### 要件定義書作成例

```
1. 新規ページ作成 → タイトル「要件定義書_プロジェクトX」
2. テンプレート「Requirements Document」を選択
3. セクション構成:
   - 1. 概要
   - 2. 目的・背景
   - 3. スコープ
   - 4. 機能要件（表形式）
   - 5. 非機能要件
   - 6. 制約事項
4. Status Macroでステータス表示（Draft/Review/Approved）
5. Jira Macroで関連課題を埋め込み
6. チームメンバーをメンション(@username)してレビュー依頼
```

### 設計書作成例

```
1. スペース「プロジェクトX」配下にページ作成
2. タイトル「システム設計書」
3. Table of Contentsマクロで目次自動生成
4. 子ページとして詳細設計を作成:
   - データベース設計
   - API設計
   - インフラ設計
5. Code Blockマクロでコード例を挿入
6. Lucidchart/Draw.io統合で図表追加
7. バージョン履歴でレビュー履歴を管理
```

## メリット

### ✅ 主な利点

1. **Jira完全統合**: 課題とドキュメントをシームレスに連携
2. **テンプレート豊富**: 会議議事録、計画書、要件定義書等
3. **バージョン管理**: 変更履歴の自動保存・復元
4. **強力な検索**: 全文検索と関連コンテンツ提案
5. **マクロ機能**: Jira、図表、コードブロック等の埋め込み
6. **権限管理**: スペース/ページレベルでの細かい制御
7. **外部ツール統合**: Slack、Google Drive、GitHub等
8. **リアルタイム協業**: 同時編集とコメント機能
9. **モバイル対応**: iOS/Androidアプリで外出先でも編集
10. **無料プラン**: 10ユーザーまで無料で利用可能

## デメリット

### ❌ 制約・課題

1. **エディタやや使いにくい**: 最新のマークダウンエディタと比較すると劣る
2. **検索機能改善余地**: 大規模スペースでは精度が低下
3. **動作やや重い**: ページ数が増えると読み込みが遅い
4. **有料版必要**: 大規模チームではコストがかかる
5. **Atlassian依存**: Jira/Trello等との統合が前提の設計
6. **オフライン機能限定**: ブラウザベースのため制約あり
7. **日本語ドキュメント少ない**: 公式は英語中心
8. **エクスポート制限**: 複雑なページ構造はエクスポートで崩れる

## 代替ツール

| ツール | 特徴 | 比較 |
|--------|------|------|
| **Notion** | モダンUI、データベース機能 | Confluenceより柔軟だがJIRA統合弱い |
| **Microsoft SharePoint** | Microsoft 365統合 | Confluenceより複雑だが企業標準 |
| **Google Workspace (Docs)** | 無料、リアルタイム協業 | Confluenceより機能限定的 |
| **Markdown Wiki (GitLab/GitHub)** | Git管理、バージョン管理強力 | 開発者向け、非技術者には不向き |
| **Docusaurus** | 静的サイト生成、高速 | ドキュメント公開に特化 |
