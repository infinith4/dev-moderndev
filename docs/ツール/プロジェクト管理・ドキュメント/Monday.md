# monday.com

## 概要

monday.com（モンデードットコム）は、イスラエル発のクラウドベースの Work OS（Work Operating System）プラットフォームです。2012年に dapulse として創業し、2017年に monday.com に改名。カラフルで直感的なビジュアルと高い柔軟性により、プロジェクト管理、CRM、マーケティング、開発など、あらゆる業務プロセスを一元管理できます。ノーコードでカスタマイズ可能なボード、自動化、統合機能により、全世界で15万社以上の企業に利用されています。

## 主な特徴

### 1. Work OS（ワークOS）
- 単なるプロジェクト管理ツールではなく、業務基盤
- あらゆる業務プロセスに対応
- ノーコードでカスタマイズ
- 部門横断的な可視化

### 2. ビジュアル重視のUI
- カラフルで直感的なデザイン
- ステータスを色で視覚化
- 複数のビュー（カンバン、ガント、カレンダー、マップ等）
- ドラッグ&ドロップ操作

### 3. 柔軟なカスタマイズ
- 200以上のテンプレート
- カスタムカラム作成
- 自動化ルール
- カスタムダッシュボード

### 4. 自動化機能
- ノーコード自動化
- 250,000以上の自動化レシピ
- カスタム自動化
- 統合連携の自動化

### 5. 統合エコシステム
- 72以上のアプリ統合
- Slack、Teams、Gmail、Outlook
- Zoom、Dropbox、Google Drive
- Zapier、Make（Integromat）

## 基本的な使い方

### 1. ワークスペースとボードの作成

```
1. monday.com にログイン
2. 左サイドバーの「+」アイコン
3. 「新しいボード」を選択
4. テンプレートを選択または空白から開始
   - プロジェクト管理
   - CRM
   - タスク管理
   - マーケティング
   - 開発スプリント
5. ボード名を入力
6. 「作成」をクリック
```

### 2. アイテム（タスク）の追加

```
1. ボードを開く
2. グループ内の「+ アイテムを追加」をクリック
3. アイテム名を入力
4. Enter キーで作成
5. 各カラムをクリックして詳細を入力
   - ステータスを選択（色分け）
   - 担当者を選択
   - 期限を設定
   - 優先度を選択
```

### 3. ビューの切り替え

```
# カンバンビュー
1. ボード上部の「+」→「カンバン」
2. グループ化するカラムを選択（例: ステータス）
3. カードをドラッグ&ドロップで移動

# ガントチャート
1. 「+」→「ガント」
2. タイムラインカラムを選択
3. バーをドラッグして期間を調整
4. 依存関係を設定

# カレンダー
1. 「+」→「カレンダー」
2. 日付カラムを選択
3. 月・週・日表示を切り替え
```

### 4. 自動化の設定

```
1. ボード右上の「自動化」ボタン
2. 「自動化を作成」をクリック
3. トリガーとアクションを選択

# 例1: ステータス変更時に担当者に通知
トリガー: ステータスが「完了」に変わったとき
アクション: 担当者に通知

# 例2: 期限超過アイテムを自動移動
トリガー: 期限が過ぎたとき
アクション: アイテムを「遅延」グループに移動

# 例3: 繰り返しタスクの自動作成
トリガー: 毎週月曜日
アクション: 新しいアイテムを作成
```

### 5. ダッシュボードの作成

```
1. 左サイドバーの「ダッシュボード」
2. 「新しいダッシュボード」
3. ウィジェットを追加
   - チャート（棒・円・折れ線）
   - 数値（KPI）
   - タイムライン
   - バッテリー（進捗）
   - ボードビュー
4. データソース（ボード）を選択
5. フィルター・グループ化を設定
6. レイアウトを調整
```

## 高度な機能

```
# Slack 統合
1. 統合センターから「Slack」を選択
2. ワークスペースを認証
3. 通知チャンネルを設定
4. 通知イベントを選択
   - アイテムの作成・更新

   - 期限通知

# Gmail 統合
1. 「Gmail」統合を有効化
2. メールからアイテムを作成
3. アイテムにメールを添付
4. アイテム更新をメール通知

# Google Calendar 統合
1. 「Google Calendar」統合
2. カレンダーを選択
3. アイテムの期限がカレンダーに同期
4. 双方向同期
```

### 数式カラム

```
# 基本的な計算
{予算} - {実績}

# 条件分岐
IF({ステータス} = "完了", "完了", "未完了")

# 日付計算
DAYS({期限}, TODAY())

# 合計
SUM({サブアイテム.工数})

# 例: 進捗率の計算
ROUND(({完了タスク数} / {総タスク数}) * 100)
```

### サブアイテム

```
1. アイテムをクリック
2. 「サブアイテムを追加」
3. サブアイテム名を入力
4. 独自のカラム構成を設定

# 例: 開発タスク
親アイテム: ログイン機能の実装
├── サブアイテム: UI設計
├── サブアイテム: API実装
├── サブアイテム: フロントエンド実装
└── サブアイテム: テスト

# サブアイテムの集計
親アイテムで数式カラム使用
SUM({サブアイテム.工数})
```

### 依存関係

```
1. ガントビューを開く
2. アイテムの端から別のアイテムにドラッグ
3. 依存関係が設定される
4. 先行タスクが遅れると後続タスクも自動調整

# 依存関係のタイプ
- FS（Finish to Start）: 完了後に開始
- SS（Start to Start）: 同時開始
- FF（Finish to Finish）: 同時完了
```

### ボード間連携

```
# 連携カラムの使用
1. カラムを追加 → 「連携」
2. 連携先ボードを選択
3. アイテムをリンク
4. 連携先のデータを表示・更新

# ミラーカラム
連携カラムで接続したボードのカラムを表示

# 例: プロジェクト管理 ⇔ タスク管理
プロジェクトボード → タスクボードへリンク
タスクの進捗をプロジェクトボードに自動反映
```

### テンプレート作成

```
1. ボードを標準化
2. ボードメニュー → 「テンプレートとして保存」
3. テンプレート名・説明を入力
4. 公開範囲を設定（プライベート/チーム/パブリック）
5. 次回から「テンプレートから作成」で使用

# テンプレートに含まれる要素
- カラム構成
- グループ
- 自動化ルール
- サンプルアイテム
- ビュー設定
```

## API 利用

### REST API

```javascript
// Node.js での使用
const mondaySDK = require('monday-sdk-js');
const monday = mondaySDK();

monday.setToken('YOUR_API_TOKEN');

// ボード一覧の取得
monday.api(`query {
  boards {
    id
    name
  }
}`).then(res => {
  console.log(res);
});

// アイテムの作成
monday.api(`mutation {
  create_item (
    board_id: 123456789,
    group_id: "topics",
    item_name: "新しいタスク",
    column_values: "{\\"status\\":\\"Working on it\\",\\"person\\":\\"12345678\\"}"
  ) {
    id
  }
}`).then(res => {
  console.log(res);
});

// アイテムの更新
monday.api(`mutation {
  change_column_value (
    board_id: 123456789,
    item_id: 987654321,
    column_id: "status",
    value: "{\\"label\\":\\"Done\\"}"
  ) {
    id
  }
}`);
```

### Python での使用

```python
import requests

API_URL = "https://api.monday.com/v2"
API_TOKEN = "YOUR_API_TOKEN"

headers = {
    "Authorization": API_TOKEN,
    "Content-Type": "application/json"
}

# アイテム一覧の取得
query = '''
query {
  boards(ids: 123456789) {
    items {
      id
      name
      column_values {
        id
        text
      }
    }
  }
}
'''

response = requests.post(API_URL, json={"query": query}, headers=headers)
data = response.json()
print(data)
```

## まとめ

monday.com は、以下の場面で特に有用です:

1. **ビジュアル重視** - カラフルで直感的なUIによる可視化
2. **柔軟なカスタマイズ** - ノーコードであらゆる業務プロセスに対応
3. **強力な自動化** - 業務効率化と人的ミスの削減

プロジェクト管理だけでなく、CRM、マーケティング、HR、IT、運用など、あらゆる部門の業務を一元管理できる柔軟性が最大の特徴です。
