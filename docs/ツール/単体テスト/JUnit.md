# JUnit

## 概要
JUnit は Java 向けの代表的な単体テストフレームワークです。テスト構造が明確で、IDE と CI の双方で安定運用しやすいのが特徴です。

## 料金
| プラン | 料金 | 備考 |
|---|---:|---|
| OSS | 無料 | Java エコシステムで広く採用 |

## 主な特徴
| 特徴 | 内容 |
|---|---|
| 標準性 | Java 開発で事実上の標準 |
| 拡張性 | パラメータ化テストや拡張機構を利用可能 |
| 統合性 | Maven/Gradle/IDE/CI と連携しやすい |

## 主な機能
| 機能 | 用途 | 実務での使いどころ |
|---|---|---|
| 単体テスト | 振る舞い検証 | サービス/ユーティリティ検証 |
| パラメータ化 | ケース展開 | 境界値や入力網羅 |
| ライフサイクル管理 | 前後処理統制 | テスト初期化の標準化 |

## インストールとセットアップ
1. ビルドツールに JUnit 依存を追加します。
2. テストディレクトリ構成と命名規約を統一します。
3. CI でテスト結果レポートを保存します。

## 基本的な使い方
1. 1テスト1責務でケースを設計します。
2. 例外系と境界値を優先して追加します。
3. 失敗時ログを見て原因を分離し、修正後に再実行します。

| 観点 | 内容 |

## デメリット
| 観点 | 内容 |
|---|---|
| 学習 | 拡張機能まで使うと理解が必要 |
| 記述量 | 小規模案件では冗長になりやすい |
| モック連携 | 外部ライブラリ選定が必要 |

## 他ツールとの比較
| ツール | 強み | 使い分け |
|---|---|---|
| JUnit | Java 標準として安定 | Java 全般 |
| TestNG | 高度なテスト設定 | 複雑な実行制御が必要な場合 |
| Spock | 表現力が高い | Groovy ベース運用 |

## ベストプラクティス
1. テスト名に前提・操作・期待結果を含めます。
2. 依存注入を活用して外部依存を分離します。
3. 単体テスト失敗をリリースゲートに設定します。

## まとめ

1. ** 検証効率 ** : JUnit は Java 単体テストの基盤として信頼性が高い選択です。
2. ** 自動運用 ** : 継続運用では命名規約と失敗時調査手順の標準化が効果的です。
3. ** 実務利点3 ** : モックツールと組み合わせることで外部依存の影響を抑えられます。
