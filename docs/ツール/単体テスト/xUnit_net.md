# xUnit.net

## 概要
xUnit.net は .NET 向けの単体テストフレームワークです。モダン .NET 開発に適した設計で、並列実行や拡張性に優れています。

## 料金
| プラン | 料金 | 備考 |
|---|---:|---|
| OSS | 無料 | .NET エコシステムで利用可能 |

## 主な特徴
| 特徴 | 内容 |
|---|---|
| .NET 親和性 | SDK と CI に統合しやすい |
| 実行効率 | 並列実行で高速化しやすい |
| 拡張性 | fixture や属性で運用を整理しやすい |

## 主な機能

### コア機能

| 機能 | 用途 | 実務での使いどころ |
|---|---|---|
| 単体テスト | 振る舞い検証 | 回帰防止 |
| データ駆動 | ケース展開 | 境界値網羅 |
| fixture 管理 | 共有前提の制御 | 初期化コスト最適化 |

### 連携機能

| 機能 | 説明 |
|------|------|
| ツール連携 | 既存の開発・運用ツールと連携しやすい |
| 自動実行 | ローカル実行と CI 実行の両方に対応しやすい |

### 運用機能

| 機能 | 説明 |
|------|------|
| 結果管理 | 実行結果を蓄積して比較しやすい |
| 変更追跡 | 設定変更の影響を確認しやすい |

## インストールとセットアップ

公式URL:
- [公式リンク1](https://xunit.net/)
- [公式リンク2](https://xunit.net/docs/getting-started/netcore/cmdline)

セットアップ手順:
1. テストプロジェクトを作成し、依存を整理します。
2. 命名規約とテスト分類方針を定義します。
3. CI で実行と結果レポート保存を設定します。

## 基本的な使い方
1. サービスやユーティリティ単位でケースを作成します。
2. 外部依存は Moq などで分離します。
3. 失敗時はログと入力条件を確認し、再現手順を固定します。

最小実行例:
- ローカル環境で xUnit.net の基本機能を実行
- CI で定期実行し、結果を比較

## 他ツールとの比較
| ツール | 強み | 使い分け |
|---|---|---|
| xUnit.net | モダン .NET で扱いやすい | 標準採用候補 |
| NUnit | 歴史が長く機能が豊富 | 既存資産継続 |
| MSTest | Microsoft 公式基盤 | 企業標準環境 |

## ベストプラクティス
1. テスト分類を明確化し、長時間ケースを分離します。
2. 外部依存の境界を設計で明確にします。
3. 実装詳細より公開契約を中心に検証します。
