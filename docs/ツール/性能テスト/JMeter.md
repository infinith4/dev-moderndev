# JMeter

## 概要
Apache JMeter は Web/API を中心に負荷試験や応答時間測定を行う性能テストツールです。GUI での作成と CLI 実行の両方に対応し、導入しやすいのが特徴です。

## 料金
| プラン | 料金 | 備考 |
|---|---:|---|
| OSS | 無料 | Apache License で利用可能 |

## 主な特徴
| 特徴 | 内容 |
|---|---|
| GUI 作成 | テスト計画を視覚的に組み立てられる |
| プラグイン拡張 | 集計や可視化を用途に応じて拡張可能 |
| 実行形態 | ローカル・分散・CI 実行に対応 |

## 主な機能
| 機能 | 用途 | 実務での使いどころ |
|---|---|---|
| 負荷試験 | 同時接続やスループット評価 | リリース判定 |
| アサーション | 応答内容の妥当性確認 | 性能と機能の同時確認 |
| レポート | 応答時間分布の分析 | ボトルネック特定 |

## インストールとセットアップ
1. Java 実行環境を準備し、JMeter を導入します。
2. テスト対象の API/URL、認証、データ条件を整理します。
3. 性能目標（p95、エラー率、TPS）を設定します。

## 基本的な使い方
1. テスト計画でリクエスト、ユーザー数、ランプアップを設定します。
2. 試験を実行し、応答時間と失敗率を確認します。
3. 結果を比較して、改善対象を優先順位付けします。

## メリット
| 観点 | 内容 |
|---|---|

## デメリット
| 観点 | 内容 |
|---|---|
| 保守性 | GUI 中心だと差分管理が難しくなる |
| 実行負荷 | 大規模試験では実行基盤の調整が必要 |
| 品質管理 | シナリオが複雑化すると属人化しやすい |

## 他ツールとの比較
| ツール | 強み | 使い分け |
|---|---|---|
| JMeter | GUI と実績が豊富 | 幅広いチームでの標準化 |
| Gatling | 高負荷・分析が強い | 継続的な性能検証 |
| k6 | スクリプト管理がしやすい | CI/CD 主体運用 |
| Locust | Python で柔軟 | Python 開発文化と相性が良い |

## ベストプラクティス
1. GUI で作成後は再利用しやすい形で計画を整理します。
2. 閾値ベースの合否判定を設定し、主観的判断を避けます。
3. 監視メトリクスと合わせて分析し、原因切り分けを早めます。
