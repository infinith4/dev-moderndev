# Bizagi Modeler

## 概要

Bizagi Modeler は BPMN 2.0 に準拠した業務プロセスモデリングツールである。業務フローの可視化、改善検討、関係者レビューを行う際に使いやすい。

## 料金

| 区分 | 内容 |
|------|------|
| Desktop版 | 無料 |
| 補足 | 連携製品や周辺サービスは有償プランあり |

## 主な特徴

| 項目 | 内容 |
|------|------|
| BPMN 特化 | BPMN 2.0 記法を標準で利用可能 |
| モデル可読性 | 業務部門とIT部門で共通理解しやすい |
| 共有しやすさ | ドキュメント出力でレビューを回しやすい |
| 導入容易 | 小規模プロセスから段階的に導入可能 |
| 教育しやすい | 記法ルールを統一しやすい |

## 主な機能

### モデリング機能

| 機能 | 説明 |
|------|------|

| サブプロセス | 複雑手順を分割して管理 |

### 運用機能

| 機能 | 説明 |
|------|------|

## インストールとセットアップ

公式URL:
- [Bizagi Modeler 公式](https://www.bizagi.com/en/platform/modeler)
- [Bizagi ドキュメント](https://help.bizagi.com/)

セットアップ手順:
1. Bizagi Modeler をインストールする。
2. チームで BPMN 記法ルールと命名規則を決める。
3. まず主要業務1つでモデル化し、レビュー運用を確立する。

## 基本的な使い方

1. 対象業務の開始条件・終了条件を先に定義する。
2. タスクと判断分岐を BPMN で整理する。
3. レーンで担当部門を明確化する。
4. レビューで用語と手順を統一し、確定版を配布する。

## メリット

- 業務プロセスを標準記法で可視化できる
- 部門間の認識差異を減らしやすい
- 改善前後の比較がしやすい
- 文書化とレビューを効率化できる

## デメリット

- BPMN の基本学習が必要
- 運用ルールがないと図の品質がばらつく
- 大規模プロセスは分割設計が必須

## 他ツールとの比較

| ツール | 主な用途 | 特徴 |
|------|------|------|
| Bizagi Modeler | BPMN モデリング | BPMN 特化で業務可視化しやすい |
| Draw.io | 汎用作図 | 自由度が高いが記法統制は手動 |
| Lucidchart | 共同作図 | クラウド協業に強い |
| PlantUML | テキスト図生成 | バージョン管理と相性が良い |

## ベストプラクティス

### 1. 記法ルールを固定

- 命名、色、粒度をチームで統一する
- 例外ルールを最小化する

### 2. 粒度を揃える

- 1図で扱う範囲を明確にする
- 複雑化したらサブプロセスへ分割する

### 3. 定期レビューを実施

- 実運用との差分を定期点検する
- 変更理由を記録して追跡可能にする
