# Draw.io (diagrams.net)

## 概要

Draw.io（diagrams.net）は汎用図表作成ツールである。フローチャート、UML、ER図、ネットワーク図などを無料で作成でき、設計資料の初期作成から共有まで幅広く使える。

## 料金

| 区分 | 内容 |
|------|------|
| 本体 | 無料 |
| 補足 | クラウド保存先の利用料は別途の場合あり |

## 主な特徴

| 項目 | 内容 |
|------|------|
| 無料利用 | 導入コストを抑えやすい |
| 汎用性 | 幅広い図種に対応 |
| 使いやすさ | GUI で直感的に作図可能 |
| 保存柔軟性 | ローカル/クラウド保存に対応 |
| 共有容易 | PNG/PDF/SVG などで出力可能 |

## 主な機能

### 作図機能

| 機能 | 説明 |
|------|------|
| テンプレート | 図種ごとのひな型を利用可能 |
| ライブラリ | UML、ER、ネットワーク図形を利用可能 |
| 接続・整列 | 図の可読性を保ちやすい |
| レイヤー管理 | 複雑図の整理に有効 |

### 出力・共有機能

| 機能 | 説明 |
|------|------|
| 形式出力 | PNG/PDF/SVG/XML で出力 |
| 埋め込み | ドキュメントへの貼り付けが容易 |
| 共同編集 | 保存先次第で協業運用可能 |
| バックアップ | XML 保持で再編集しやすい |

### 運用機能

| 機能 | 説明 |
|------|------|
| 命名規則運用 | 図ファイル管理を標準化しやすい |
| 版管理連携 | Git 等で変更履歴管理可能 |
| レビュー支援 | 差分確認をしやすい出力運用が可能 |
| 教育用途 | 設計説明資料として扱いやすい |

## インストールとセットアップ

公式URL:
- [diagrams.net 公式](https://www.diagrams.net/)
- [利用ガイド](https://www.drawio.com/doc/)

セットアップ手順:
1. ブラウザ版またはデスクトップ版を選択する。
2. 保存先（ローカル/クラウド）と命名規則を決める。
3. チーム標準テンプレートを用意して運用開始する。

## 基本的な使い方

1. 図の目的（説明対象・読者）を先に定義する。
2. テンプレートから最小構成で作図を始める。
3. 役割色分けや凡例を入れて可読性を上げる。
4. 出力形式を統一し、レビューしやすい形で共有する。

## メリット

- 無料で導入しやすい
- 図の種類を横断して使える
- 学習コストが比較的低い
- 設計資料作成スピードが上がる

## デメリット

- 記法統制をしないと図品質がばらつく
- 大規模図は構造設計が必要
- 同時編集運用は保存先依存になる

## 他ツールとの比較

| ツール | 主な用途 | 特徴 |
|------|------|------|
| Draw.io | 汎用作図 | 無料・汎用で導入しやすい |
| Lucidchart | クラウド作図 | 協業機能が強い |
| Visio | 業務図作成 | Office 連携に強い |
| Mermaid | テキスト作図 | ドキュメント連携がしやすい |

## ベストプラクティス

### 1. テンプレートを標準化

- 図種ごとにひな型を用意する
- 色や凡例のルールを固定する

### 2. ファイル運用を統一

- 命名規則と保存先を決める
- 版管理可能な形式を保持する

### 3. レビュー観点を定義

- 読み手視点で情報過多を避ける
- 変更理由を図更新時に記録する
