# PlantUML

## 概要

PlantUML はテキスト記法で UML や各種図を生成するツールである。図をコードとして扱えるため、設計変更の追跡やレビュー運用に適している。

## 料金

| 区分 | 内容 |
|------|------|
| PlantUML 本体 | 無料（OSS） |
| 補足 | 実行環境や周辺連携ツールの費用は別途の場合あり |

## 主な特徴

| 項目 | 内容 |
|------|------|
| UML 向き | クラス図、シーケンス図などに強い |
| テキスト管理 | バージョン管理と相性が良い |
| 自動生成 | 手作業レイアウトを減らせる |
| 連携性 | IDE、CI、ドキュメント基盤と連携可能 |
| 再利用性 | テンプレート化しやすい |

## 主な機能

### 図生成機能

| 機能 | 説明 |
|------|------|
| UML 図生成 | クラス、シーケンス、ユースケース等を作成 |
| 非 UML 図 | ER、Gantt、状態遷移等にも対応 |
| include 利用 | 共通定義の再利用が可能 |
| スキン調整 | 体裁を一定範囲で統一可能 |

### 連携機能

| 機能 | 説明 |
|------|------|
| IDE 連携 | 編集時プレビューを活用可能 |
| ドキュメント連携 | Markdown 等へ組み込みやすい |
| CI 連携 | 図生成を自動化可能 |
| 画像出力 | PNG/SVG 等で共有可能 |

### 運用機能

| 機能 | 説明 |
|------|------|
| 版管理 | 設計差分レビューをしやすい |
| テンプレート | 図記法の標準化に有効 |
| 命名統一 | 要素命名を揃えやすい |
| 保守性 | 仕様変更へ追従しやすい |

## インストールとセットアップ

公式URL:
- [PlantUML 公式](https://plantuml.com/)
- [PlantUML ドキュメント](https://plantuml.com/ja/)

セットアップ手順:
1. 実行方法（ローカル/サーバー）を決める。
2. IDE プラグインまたはレンダリング環境を準備する。
3. 図テンプレートと命名ルールを定義する。

## 基本的な使い方

1. 図の目的と対象読者を先に決める。
2. 最小記法で構造を作り、必要要素を追加する。
3. 共通定義を include 化して重複を減らす。
4. 設計変更時は図と仕様を同時更新する。

## メリット

- UML 図をテキストで管理できる
- 設計差分レビューをしやすい
- CI で図生成を自動化できる
- テンプレート化で運用を標準化しやすい

## デメリット

- 記法学習が必要
- 複雑図は可読性維持に工夫が必要
- 表示環境依存の調整が必要な場合がある

## 他ツールとの比較

| ツール | 主な用途 | 特徴 |
|------|------|------|
| PlantUML | テキスト UML | UML 記述と版管理に強い |
| Mermaid | テキスト作図 | Markdown 連携が容易 |
| Draw.io | GUI 作図 | 視覚操作で柔軟に作図可能 |
| Visio | 業務図作成 | Office 連携に強い |

## ベストプラクティス

### 1. 図テンプレートを共通化

- 図種ごとの雛形を定義する
- 色・命名ルールを統一する

### 2. 図の責務を分離

- 1図で複数目的を持たせない
- 詳細図と概要図を分ける

### 3. 変更追跡を徹底

- 設計変更と図更新を同一PRで管理する
- 変更理由をレビューで残す

## まとめ

1. **管理性** : PlantUML は UML のテキスト管理に強い。
2. **品質力** : テンプレートと責務分離で図品質を維持しやすい。
3. **追跡性** : 設計変更と図更新を同時運用することが重要。
