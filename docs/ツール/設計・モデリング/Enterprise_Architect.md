# Enterprise Architect

## 概要

Enterprise Architectは、Sparx Systems社が開発したエンタープライズ向けUMLモデリングツールです。UML 2.5完全対応、BPMN、ArchiMate、SysMLなど多様なモデリング記法をサポートし、要件管理からシステム設計、実装、テストまでのソフトウェア開発ライフサイクル全体をカバーします。大規模プロジェクトでの利用に適した、業界標準のモデリングプラットフォームです。

## 主な機能

### 1. 包括的なモデリング対応
- **UML 2.5**: 全ダイアグラムタイプ完全対応
  - 構造図（クラス図、コンポーネント図、配置図等）
  - 振る舞い図（シーケンス図、アクティビティ図、ステートマシン図等）
- **BPMN 2.0**: ビジネスプロセスモデリング
- **ArchiMate 3.x**: エンタープライズアーキテクチャ
- **SysML**: システムエンジニアリング
- **TOGAF、Zachman**: フレームワークサポート

### 2. 要件管理
- 要件の定義・トレーサビリティ管理
- ユースケースとの関連付け
- 要件マトリクス自動生成

### 3. コード生成・リバースエンジニアリング
- **コード生成**: Java、C++、C#、Python、VB.NETなど
- **リバースエンジニアリング**: 既存コードからモデル生成
- **ラウンドトリップ**: モデル↔コードの双方向同期

### 4. チームコラボレーション
- 共有リポジトリ（SQL Server、MySQL、Oracle、PostgreSQL）
- バージョン管理（SVN、Git統合）
- ベースライン管理
- モデルレビュー・コメント機能

### 5. ドキュメント生成
- RTF、PDF、HTML形式での自動ドキュメント生成
- カスタマイズ可能なテンプレート
- 設計書自動作成

### 6. シミュレーション・分析
- モデルシミュレーション
- メトリクス分析
- 影響分析（変更影響の可視化）

## 利用方法

### 基本的な使い方

1. **プロジェクト作成**
   ```
   File → New Project
   → プロジェクト名・保存先を指定
   → リポジトリタイプ選択（ローカル/共有DB）
   ```

2. **パッケージ作成**
   ```
   Project Browser → 右クリック → Add Package
   → パッケージ名入力（例: ドメインモデル）
   ```

3. **ダイアグラム作成**
   ```
   パッケージを右クリック → Add Diagram
   → ダイアグラムタイプ選択（クラス図、シーケンス図等）
   ```

4. **要素配置**
   ```
   Toolboxから要素をドラッグ&ドロップ
   → プロパティウィンドウで属性・操作を定義
   → 関連線で要素間を接続
   ```

5. **コード生成**
   ```
   クラスを選択 → Code Engineering → Generate Code
   → 言語選択（Java/C#等）
   → 出力先ディレクトリ指定
   ```

### クラス図作成例

```
1. 新規クラス図を追加
2. Toolboxから「Class」を複数配置
3. 各クラスの属性（フィールド）を定義
   - 例: - name: String, - age: int
4. 各クラスの操作（メソッド）を定義
   - 例: + getName(): String
5. 関連（Association）、汎化（Generalization）で接続
6. カーディナリティ（多重度）を設定
7. コード生成でJava/C#コードを出力
```

### シーケンス図作成例

```
1. 新規シーケンス図を追加
2. Toolboxから「Lifeline」を配置（アクター、システム等）
3. メッセージを矢印で接続
   - 同期メッセージ（実線矢印）
   - 非同期メッセージ（破線矢印）
4. 活性化ボックスで処理期間を表現
5. 条件分岐（alt）、ループ（loop）フラグメント追加
```

## メリット

### ✅ 主な利点

1. **UML 2.5完全対応**: すべてのUMLダイアグラムをサポート
2. **多様なモデリング記法**: BPMN、ArchiMate、SysML等
3. **エンタープライズ実績**: 大規模プロジェクトでの豊富な採用実績
4. **コード生成**: モデルからソースコード自動生成
5. **要件トレーサビリティ**: 要件と設計の完全な追跡
6. **共有リポジトリ**: チームでのモデル共有・バージョン管理
7. **ドキュメント自動生成**: 設計書をワンクリックで作成
8. **買い切りライセンス**: サブスクではなく永続ライセンス
9. **カスタマイズ性**: アドイン、スクリプトで機能拡張可能
10. **長期サポート**: 20年以上の開発・サポート実績

## デメリット

### ❌ 制約・課題

1. **学習曲線急**: 機能が豊富なため習得に時間がかかる
2. **UI古い**: 最新のモダンUIツールと比較すると見劣り
3. **Windows中心**: Mac版は機能制限あり
4. **高額**: エンタープライズ向けエディションは高価
5. **動作やや重い**: 大規模モデルでは動作が遅い
6. **日本語ドキュメント少ない**: 公式ドキュメントは英語中心
7. **クラウド機能弱い**: ブラウザベースツールと比較して劣る
8. **セットアップ複雑**: 共有リポジトリ設定にデータベース知識必要

## 代替ツール

| ツール | 特徴 | 比較 |
|--------|------|------|
| **PlantUML** | テキストベースUML、無料 | EA比で機能限定的だが軽量 |
| **Visual Paradigm** | 類似機能、やや低価格 | EAと同等機能、UI改善 |
| **astah*** | 日本製、日本語完全対応 | EAより軽量、機能はやや少ない |
| **StarUML** | モダンUI、低価格 | エンタープライズ機能はEAが上 |
| **Lucidchart** | クラウドベース、協業重視 | UML特化ではないが使いやすい |
