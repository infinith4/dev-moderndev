# Zammad

## 概要

Zammad はオープンソースのヘルプデスク/チケット管理システムである。メール、Web、チャットなど複数チャネルを統合し、問い合わせ対応の標準化と可視化を行いやすい。

## 料金

| 区分 | 内容 |
|------|------|
| Community（セルフホスト） | 無料（OSS） |
| Hosted プラン | エージェント課金（プラン別） |

## 主な特徴

| 項目 | 内容 |
|------|------|
| OSS 運用 | セルフホストで柔軟に運用可能 |
| マルチチャネル | メール・チャット・SNS 等を統合管理 |
| モダン UI | 日常オペレーションを扱いやすい |
| API 対応 | 外部システムとの連携が可能 |
| 導入しやすさ | Docker 構成で比較的短期導入可能 |

## 主な機能

### チケット運用機能

| 機能 | 説明 |
|------|------|
| チケット管理 | 受付、割当、進捗、クローズを一元管理 |
| 自動分類 | ルールでタグ・優先度・担当を設定 |
| テンプレート返信 | 定型対応を標準化 |
| 重複統合 | 重複問い合わせをマージ可能 |

### サポート機能

| 機能 | 説明 |
|------|------|
| ナレッジ管理 | FAQ と解決手順を蓄積 |
| SLA 管理 | 応答/解決目標を追跡 |
| 検索機能 | 過去対応の再利用を促進 |
| レポート | 対応件数・遅延傾向を可視化 |

### 連携・運用機能

| 機能 | 説明 |
|------|------|
| LDAP/SSO | 認証基盤と統合可能 |
| Webhook/API | 他ツール連携を実装可能 |
| 権限管理 | ロール別アクセス制御 |
| デプロイ柔軟性 | セルフホスト/Hosted を選択可能 |

## インストールとセットアップ

公式URL:
- [Zammad 公式](https://zammad.com/)
- [Zammad Docs](https://docs.zammad.org/)
- [Zammad GitHub](https://github.com/zammad/zammad)

セットアップ手順:
1. セルフホストか Hosted かを運用方針で決める。
2. チケット分類、優先度、SLA の基本ルールを先に定義する。
3. メールチャネル連携と担当グループ割当を設定する。
4. ナレッジとテンプレート返信を整備して運用を開始する。

## 基本的な使い方

1. まず主要問い合わせカテゴリを定義する。
2. 受信チャネルごとに自動割当ルールを設定する。
3. SLA 違反通知を有効化して遅延対応を防ぐ。
4. 定期的にナレッジ更新とルール見直しを行う。

## メリット

- OSS で導入しやすい
- マルチチャネル対応で運用を集約しやすい
- UI が分かりやすく定着しやすい
- API 連携で拡張しやすい

## デメリット

- 大規模組織では追加設計が必要になりやすい
- ITIL フル運用には専用 ITSM より機能が限定的
- セルフホスト時はインフラ運用負担が発生する

## 他ツールとの比較

| ツール | 主な用途 | 特徴 |
|------|------|------|
| Zammad | ヘルプデスク運用 | OSS で柔軟、導入しやすい |
| ServiceNow | エンタープライズ ITSM | 統合機能が豊富 |
| Jira Service Management | ITSM/開発連携 | DevOps 連携に強い |
| Freshdesk | クラウドサポート | SaaS で短期導入しやすい |

## ベストプラクティス

### 1. 運用ルールを先に決める

- 優先度定義とエスカレーション条件を明文化する
- 担当グループと責任範囲を固定する

### 2. ナレッジ運用を定常化

- 解決済みチケットから FAQ を継続追加する
- 古い手順を定期更新する

### 3. 自動化を小さく始める

- 自動タグ付け・割当から導入する
- 効果測定しながらルールを拡張する
