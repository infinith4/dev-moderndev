# Lucidchart（ユースケース図作成）

## 概要

Lucidchartは、クラウドベースの図表作成ツールで、UMLユースケース図、フローチャート、ER図など、様々な図表の作成に対応しています。直感的な操作性とリアルタイム協業機能により、チームでのユースケース分析作業を効率化できます。

### 主な特徴

- **ブラウザベース**: インストール不要で、ブラウザからアクセス可能
- **リアルタイム協業**: 複数ユーザーが同時に編集可能
- **UMLテンプレート**: ユースケース図をはじめとするUMLテンプレートが豊富
- **直感的操作**: ドラッグ&ドロップで簡単に図を作成
- **統合機能豊富**: Google Workspace、Microsoft 365、Slack等との連携

### 料金プラン

- **Free版**: 基本機能（ドキュメント数・図形数に制限あり）
- **Individual**: $7.95/月（個人向け、無制限ドキュメント）
- **Team**: $9/月/ユーザー（チーム向け、高度な協業機能）
- **Enterprise**: カスタム価格（大企業向け）

### メリット・デメリット

**メリット**
- ブラウザベースでどこからでもアクセス可能
- リアルタイム協業により、チームでの作業が効率的
- UMLテンプレートが豊富で、すぐにユースケース図を作成開始できる
- 直感的な操作で、UMLの専門知識がなくても図を作成可能
- 各種クラウドサービスとの統合機能が充実

**デメリット**
- 無料版は機能制限あり（ドキュメント数、図形数など）
- 厳密なUML検証機能がない（記法の誤りをチェックしない）
- コード生成機能がない
- 複雑なモデリングには不向き（シンプルな図表作成に最適）

## 利用方法

### 1. アカウント作成

1. [Lucidchart公式サイト](https://www.lucidchart.com/)にアクセス
2. 「Sign up free」をクリック
3. メールアドレスまたはGoogle/Microsoft アカウントで登録
4. アカウントを有効化

### 2. UMLユースケース図の作成

1. ダッシュボードから「+ New」ボタンをクリック
2. 「UML Use Case」テンプレートを検索
3. 適切なテンプレートを選択、または「Blank UML Use Case Diagram」を選択
4. ドキュメント名を入力（例: "予約システム ユースケース図"）

### 3. UML図形ライブラリの利用

#### 図形ライブラリの表示

1. 左側のパネルで「Shapes」をクリック
2. 「UML」カテゴリを展開
3. 「Use Case」ライブラリを選択
4. 必要に応じて他のライブラリも追加

#### 主要な図形

- **Actor**: アクター（人型アイコン）
- **Use Case**: ユースケース（楕円形）
- **System Boundary**: システム境界（矩形）
- **Association**: 関連線（実線）
- **Include/Extend**: インクルード/エクステンド（破線矢印）
- **Generalization**: 汎化（白抜き三角矢印）

### 4. アクターとユースケースの配置

#### アクターの配置

1. 左パネルの「UML Use Case」ライブラリから「Actor」をドラッグ
2. キャンバス上の適切な位置にドロップ
3. アクター名をダブルクリックして編集（例: "患者"、"医師"、"受付担当者"）

#### ユースケースの配置

1. ライブラリから「Use Case」（楕円形）をドラッグ
2. キャンバスにドロップ
3. ユースケース名を入力（例: "予約する"、"診察記録を作成する"）

#### システム境界の追加

1. ライブラリから「System Boundary」（矩形）をドラッグ
2. ユースケースを囲むように配置
3. サイズを調整して、すべてのユースケースを含める
4. システム名を入力（例: "病院予約システム"）

### 5. 関連線の接続

#### 基本的な接続方法

1. アクターまたはユースケースをクリック
2. 図形の端に表示される赤い点をクリック
3. 接続先の図形までドラッグ
4. 自動的に関連線が作成される

#### Include関係の作成

1. 左パネルから「Dependency」（破線矢印）を選択
2. 呼び出し元のユースケースから呼び出し先のユースケースへドラッグ
3. 矢印をダブルクリックして、ステレオタイプを«include»に変更

#### Extend関係の作成

1. 「Dependency」を選択
2. 拡張元のユースケースから基本ユースケースへドラッグ
3. 矢印をダブルクリックして、ステレオタイプを«extend»に変更

### 6. ユースケース図の具体例

**例: 病院予約システムのユースケース図**

#### アクター
- **患者**: 病院で診察を受ける人
- **医師**: 診察を行う医師
- **受付担当者**: 受付業務を行うスタッフ
- **システム管理者**: システムを管理する管理者

#### ユースケース（患者向け）
- **予約する**: 診察の予約を行う
  - **«include» ログイン**: 予約時に必須のログイン処理
  - **«include» 空き枠確認**: 予約可能な時間枠を確認
  - **«extend» リマインダー設定**: オプションでリマインダーを設定
- **予約をキャンセルする**: 既存の予約をキャンセル
- **診察履歴を閲覧する**: 過去の診察記録を閲覧

#### ユースケース（医師向け）
- **診察記録を作成する**: 診察内容を記録
- **処方箋を発行する**: 処方箋を作成
- **診察スケジュールを確認する**: 自分の診察予定を確認

#### ユースケース（受付担当者向け）
- **患者を登録する**: 新規患者を登録
- **予約を管理する**: 予約の確認・変更・キャンセル
- **会計処理**: 診察料金の会計

#### ユースケース（システム管理者向け）
- **ユーザーアカウント管理**: ユーザーアカウントの作成・削除
- **マスタデータ管理**: 診療科目、医師情報等のマスタデータ管理

#### 作成手順

1. **アクターの配置**
   - 左側に「患者」「医師」「受付担当者」「システム管理者」を縦に配置

2. **システム境界の作成**
   - 中央に「病院予約システム」という名前のシステム境界を配置

3. **ユースケースの配置**（システム境界内）
   - 予約する
   - ログイン
   - 空き枠確認
   - リマインダー設定
   - 予約をキャンセルする
   - 診察履歴を閲覧する
   - 診察記録を作成する
   - 処方箋を発行する
   - 診察スケジュールを確認する
   - 患者を登録する
   - 予約を管理する
   - 会計処理
   - ユーザーアカウント管理
   - マスタデータ管理

4. **関連線の接続**
   - 患者 → 予約する
   - 患者 → 予約をキャンセルする
   - 患者 → 診察履歴を閲覧する
   - 予約する → ログイン（«include»）
   - 予約する → 空き枠確認（«include»）
   - リマインダー設定 → 予約する（«extend»）
   - 医師 → 診察記録を作成する
   - 医師 → 処方箋を発行する
   - 医師 → 診察スケジュールを確認する
   - 受付担当者 → 患者を登録する
   - 受付担当者 → 予約を管理する
   - 受付担当者 → 会計処理
   - システム管理者 → ユーザーアカウント管理
   - システム管理者 → マスタデータ管理

### 7. スタイルとフォーマット

#### 図形のスタイル変更

1. 図形をクリックして選択
2. 右側のプロパティパネルでスタイルを設定:
   - **塗りつぶし色**: アクターとユースケースで色分け
   - **線の色・太さ**: 強調したい要素の線を太くする
   - **フォント**: フォントサイズや書体を変更

#### 色分けの例

- **アクター**: ライトブルー
- **ユースケース（患者向け）**: ライトグリーン
- **ユースケース（医師向け）**: ライトイエロー
- **ユースケース（受付向け）**: ライトピンク
- **ユースケース（管理者向け）**: ライトグレー
- **システム境界**: 白（線のみ）

### 8. ノートとコメントの追加

#### ノートの追加

1. 「UML」ライブラリから「Note」を選択
2. キャンバスに配置
3. ノート内容を入力（例: "予約は診察日の3日前まで可能"）
4. ノートを関連するユースケースに接続

#### コメント機能（協業用）

1. 図形を右クリック→「Comment」を選択
2. コメントを入力
3. チームメンバーがコメントに返信可能

### 9. レイアウトの整理

#### 自動整列

1. 複数の図形を選択（Shift + クリック）
2. 右クリック→「Align」→「Align Left/Center/Right」
3. または「Distribute Horizontally/Vertically」で均等配置

#### グリッド・スナップ機能

1. 「View」メニュー→「Grid」をオンにする
2. 図形を移動すると、グリッドにスナップして整列

### 10. ユースケース仕様書の作成（コメント機能）

Lucidchartでは、各ユースケースに詳細情報を追加できます。

1. ユースケース（楕円）をダブルクリック
2. 「Notes」または「Shape Data」に詳細を入力:

**例: 「予約する」ユースケースの詳細**

```
【ID】UC-001

【概要】
患者が診察の予約を行う

【事前条件】
- 患者がアカウントを持っている
- 患者がログイン済みである
- 予約可能な枠が存在する

【事後条件】
- 予約が確定し、予約番号が発行される
- 患者に予約確認メールが送信される
- 医師のスケジュールに予約が追加される

【基本フロー】
1. 患者がログインする（«include» ログイン）
2. 患者が診療科目を選択する
3. 患者が希望の医師を選択する
4. システムが空き枠を表示する（«include» 空き枠確認）
5. 患者が希望の日時を選択する
6. 患者が予約内容を確認する
7. 患者が「予約確定」ボタンをクリックする
8. システムが予約を確定し、予約番号を発行する
9. システムが予約確認メールを送信する
10. システムが予約完了画面を表示する

【代替フロー】
- 空き枠がない場合: 他の日時を提案する
- ログインに失敗した場合: エラーメッセージを表示し、再ログインを促す
```

### 11. チーム共有と協業

#### 共有方法

1. 右上の「Share」ボタンをクリック
2. 共有方法を選択:
   - **メールで招待**: メールアドレスを入力して招待
   - **リンク共有**: 共有リンクを生成
3. 権限を設定（View only / Can edit / Can comment）
4. 「Send」をクリック

#### リアルタイム協業

1. 共有後、チームメンバーが同時に編集可能
2. 各ユーザーのカーソルが色分けされて表示される
3. コメント機能でフィードバックを交換

### 12. エクスポート

#### 画像・PDF出力

1. 「File」→「Download As」を選択
2. 形式を選択:
   - **PDF**: ドキュメント用
   - **PNG**: プレゼンテーション用（高解像度）
   - **JPEG**: メール添付用
   - **SVG**: ベクター形式（再編集可能）
3. オプションを設定（ページ範囲、品質など）
4. 「Download」をクリック

#### Microsoft Office統合

1. 「File」→「Download As」→「Microsoft Word」
2. ユースケース図が埋め込まれたWordドキュメントが生成される
