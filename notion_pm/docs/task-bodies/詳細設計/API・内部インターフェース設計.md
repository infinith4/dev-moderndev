## タスク着手時
- 期間に対応を期間を入力してください。(5日を超えるタスクは分割してください)
- 見積(人日)に記載がなければ入力してください。
- ステータスを対応中に変更してください。
- 完了条件を満たしたら、ステータスをレビューに変更してください。
- 成果物を記載してください。(例: PR, SpreadsheetのURLなど)

## 対応内容
- システム内部および API の連携ポイントを洗い出す。
- エンドポイント、メソッド、パラメータ、レスポンス仕様を定義する。
- 認証、エラーコード、タイムアウトなどの共通仕様を整理する。
- 呼び出し元・呼び出し先双方の観点でレビューを実施する。

## 完了条件
- API 一覧または内部インターフェース一覧が作成されている。
- 入出力仕様とエラー仕様が明記されている。
- 認証や利用条件などの前提が整理されている。
- レビュー指摘が反映されている。

## 成果物
- API 設計書
- 内部インターフェース設計書
- エンドポイント一覧
- リクエスト・レスポンス定義
- エラーコード一覧
- レビュー記録
